# TheEpicWays Client Intelligence Architecture

## Purpose

TheEpicWays should feel human in the room and intelligent behind the scenes. The system helps Estefania serve clients better by improving preparation, facilitation, follow-up, measurement, and reuse of approved insights.

## System Loops

### 1. Client Signal Loop

Inputs:
- Discovery call notes
- Public company context
- Team size, market, culture, and leadership pressure
- Event goal, audience, constraints, and language

Outputs:
- Leadership tension map
- Workshop angle
- Proposed format
- Risks and sensitivities
- Questions to ask before the room

### 2. Offer Architecture Loop

Inputs:
- Client signal map
- TheEpicWays current offers
- Budget, urgency, audience, and desired depth

Outputs:
- Recommended offer tier
- Proposal outline
- Outcome language
- Timeline
- Deliverables

### 3. Room Design Loop

Inputs:
- Offer tier
- Audience profile
- Facilitation style
- TheEpicWays brand language

Outputs:
- Agenda
- Exercises
- Discussion prompts
- Slide structure
- Facilitation notes
- Energy checkpoints

### 4. Memory And Follow-Up Loop

Inputs:
- Session notes
- Decisions
- Questions raised
- Commitments
- Observed friction

Outputs:
- Executive summary
- Team action plan
- Manager scripts
- Owner tracker
- Follow-up email
- Next-session recommendation

### 5. Impact And Growth Loop

Inputs:
- Follow-up state
- Client feedback
- Outcome indicators
- Approved stories or anonymized themes

Outputs:
- Impact dashboard
- Case-study outline
- Newsletter/post ideas
- Event clips or talking points
- Offer improvements

## Data Boundary

Public pages may show the architecture, not private data. Client-specific memory belongs in controlled workspaces such as Notion, Google Drive, Sheets, CRM, or a later dedicated portal. Public assets must never expose client details without explicit approval.

## Agent Outputs

| Agent | Main Output |
|---|---|
| Client Signal Agent | Intake brief and tension map |
| Offer Architect | Proposal and product packaging |
| Workshop Designer | Agenda, exercises, and slides |
| Room Memory Agent | Summary, decisions, action tracker |
| Impact Analyst | Outcome dashboard and next steps |
| Content Repurposer | Approved posts, scripts, and talks |
| Client Portal Steward | Organized client workspace |

## Design Rule

Every system diagram should make the work feel easier to buy and easier to trust. If a diagram looks impressive but does not help a founder understand what happens before, during, and after a room, simplify it.
