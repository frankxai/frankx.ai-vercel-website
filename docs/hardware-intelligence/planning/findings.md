# Findings

## Current state

- `/ai-hardware` already provides the strongest first-session product loop: inputs, routing, recommendations, platform comparison, model fit, and fleet phases.
- It currently combines workstation, unified-memory, edge, and cloud concepts on one page without durable sub-routes.
- `/cloud` already exists as a cloud-platform catalog.
- `/ai-architecture/multi-cloud-comparison` already exists for deployed cloud architecture and cost structure.
- `/learn` and `/research` already have mature route systems but no hardware-specific entry point.
- The site already contains model-fit intelligence under the LLM hub and GPU-aware architecture content under AI Architecture.

## Core information-architecture insight

“AI hardware” is the umbrella discovery term, but cloud GPU is an access model rather than a device class. It belongs in the same decision journey and must be linked from the umbrella hub, while its canonical page should explain renting capacity and hand users to `/cloud` or AI Architecture for provider and deployment detail.

## Audience clusters

1. Individual builder: laptop versus desktop, RAM, GPU memory, local models.
2. Creator studio: workstation, media generation, editing, storage, cloud burst.
3. Privacy-first operator: unified-memory node, local inference, always-on agents.
4. Founder/team: home lab, routing, shared inference, staged expansion.
5. Enterprise architect: data-center GPUs, servers, networking, power, deployment topology.
6. Edge builder: accelerator modules, cameras, robotics, low-power inference.

## Content boundary

Device pages should explain capability envelopes and operating tradeoffs. Exact deals belong in a future dated offer layer with seller, VAT, region, warranty, and observed-at metadata. Until that ingestion exists, official specifications and price bands stay separated.

## Premium experience decision

Do not add separate visual gimmicks to every category. The signature interaction remains the main hub’s responsive compute constellation. Category pages should feel editorial and decisive, with one route map, a decision boundary, and an inspectable “choose this when / avoid this when” frame.
