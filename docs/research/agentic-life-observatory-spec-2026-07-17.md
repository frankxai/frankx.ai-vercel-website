# Agentic Life Observatory — Product and Evolution Spec

**Surface:** `/research/agentic-life-observatory`  
**Machine endpoint:** `/research/agentic-life-observatory/registry.json`  
**Audience:** AI architects, founders, developers, and operators selecting infrastructure for persistent multi-domain agents.

## Product job

Convert the four Agentic Life research hubs into an operating decision surface. The observatory must answer:

1. What should we build ourselves?
2. What should we integrate behind an adapter?
3. Who is a credible ecosystem partner?
4. Who competes with the broader thesis?
5. Which patterns should we learn from or monitor?

## Dominant scene

A dense editorial market map, not a marketing-card wall. The first viewport establishes the five structural failure modes, current registry size, and one primary action: inspect the market map.

## Operating loop

| Phase | Action | Receipt |
|---|---|---|
| Install | Classify and connect a candidate behind a reversible adapter. | Adapter ID, owner, version, export path |
| Test | Run contract, recall, trajectory, policy, and deletion tests. | Machine-readable audit JSON |
| Observe | Compare traces, scores, cost, failures, and human overrides. | Dashboard or exported trace set |
| Evolve | Promote, constrain, replace, or remove based on evidence. | Decision record + updated registry |

## Scoring

Every system receives a directional 0–3 editorial score for context, composition, sovereignty, verifiability, and multi-domain fit. Scores are evidence-linked and explicitly not certifications or procurement recommendations.

## Release gates

- Registry validates as JSON and contains unique IDs.
- Every item has HTTPS evidence, a verification date, all five scores, a risk, and a next action.
- All five Agentic Life hubs cross-link.
- The observatory has an exact static route and a JSON endpoint.
- Keyboard, semantic controls, mobile composition, reduced-motion safety, typecheck, build, and production smoke are verified.
- No private client, family, health, finance, swarm-bus, or credential material appears.

## Evolution policy

- **Weekly:** watch high-signal release notes; no score changes without evidence.
- **Monthly:** verify URLs, licenses, deployment claims, export paths, and integration status.
- **Quarterly:** rerun adapter/eval comparisons and reconsider build/integrate/partner/compete/watch roles.
- Registry freshness becomes a merge gate after 120 days; stale data must be refreshed or explicitly downgraded.
