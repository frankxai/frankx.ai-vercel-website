# Property Intelligence OS Work Page Scene Brief

## Surface

Route: `/work`

Brand: FrankX. Executive AI command layer, practical, sharp, commercially literate.

Audience: property owners, small operators, template buyers, and collaborators who need to understand the system quickly.

## First Read

In two seconds the visitor should understand:

- this is a premium operating system for rental properties
- it combines owner workspace, renter portal, listings, support, and agents
- it is brother-first, template-ready, and approval-gated
- the example is safe and inspectable without private renter data

## Dominant Visual Idea

A dark owner command surface, not a marketing card grid. The hero visual shows one property moving through owner decisions, renter support, and listing drafts.

## Motion Job

Motion is Track A only. It clarifies that the example has multiple operating surfaces and that the system is alive, but it never carries essential meaning.

Motion elements:

- preview tab panel transition
- subtle progress rail movement
- static reduced-motion fallback through `useReducedMotion`

No Track B scroll set-piece in v1 because the route needs clarity and deployment speed more than authored scroll choreography.

## Asset Strategy

No generated or external imagery in this pass. The visual is built from typed, inspectable UI artifacts. This avoids rights uncertainty and keeps the product mechanism as the hero.
