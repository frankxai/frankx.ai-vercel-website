# FrankX.AI V5 - Visual Wireframes & Layout Guide
*Page structure blueprints for V5 implementation*

## Layout Principles

### Core Design System
- **Maximum Content Width:** 1400px (7xl container)
- **Reading Width:** 65-75 characters (prose container, ~700px)
- **Vertical Rhythm:** 24px base unit (6 in Tailwind)
- **Section Padding:** 96px vertical (py-24)
- **Card Padding:** 24-32px internal (p-6 to p-8)
- **Typography Scale:** 16px base, 1.5-1.7 line height

### Visual Hierarchy
1. **Epic Scale (H1):** 48-96px (text-5xl to text-8xl)
2. **Section Scale (H2):** 36-48px (text-3xl to text-5xl)
3. **Subsection Scale (H3):** 24-30px (text-2xl to text-3xl)
4. **Body Scale:** 16-20px (text-base to text-lg)
5. **Detail Scale:** 12-14px (text-xs to text-sm)

## MOBILE-FIRST CONSIDERATIONS

### Breakpoints
- **Mobile:** < 640px (sm)
- **Tablet:** 640px - 1024px (md/lg)
- **Desktop:** > 1024px (xl+)

### Mobile Optimizations

```
1. Hero Sections:
   - Reduce headline size by 50% on mobile
   - Stack CTAs vertically
   - Hide background complexity
   - Show mobile-specific images

2. Grid Layouts:
   - 3-column â†’ 1-column on mobile
   - 2-column â†’ 1-column below tablet
   - Card padding: 24px â†’ 16px

3. Navigation:
   - Hamburger menu on mobile
   - Sticky header with smaller height
   - Hide secondary nav items

4. Typography:
   - Minimum 16px body text
   - Increase line height to 1.7-1.8
   - Shorter line lengths (45-55 characters)

5. Touch Targets:
   - Minimum 44px tap area for buttons
   - Increase spacing between links
   - Larger form inputs
```
