# EpicWays Intelligence Starter Kit

Version: 0.1.0
Release date: 2026-06-23
Public page: https://frankx.ai/downloads/epicways-intelligence

This is a practical starter operating system for Estefania Badra and TheEpicWays. It helps her prepare better rooms, design clearer offers, follow up with more care, and track what changed for clients.

It is not a replacement for her voice, judgment, facilitation, or relationships. The system sits behind the work so TheEpicWays can serve clients with more preparation, memory, and consistency.

## What It Helps With

- Read a client situation before the room.
- Turn a messy request into a clean offer.
- Build a workshop or event flow faster.
- Capture decisions, owners, and follow-up.
- Track practical change after the session.
- Reuse approved insights for proposals, talks, posts, and future programs.

## The Five Loops

1. Signal - capture context, team tension, goals, audience, constraints, and language.
2. Shape - package the right offer: diagnostic, workshop, lab, event, or retainer.
3. Room - design agenda, exercises, prompts, slides, and facilitation notes.
4. Memory - turn the room into summaries, actions, owner trackers, and next steps.
5. Growth - reuse approved themes to improve content, offers, events, and future delivery.

## Quick Start

1. Copy this folder into a private workspace.
2. Start with `01-client-signal-brief.md` for a real prospect or client.
3. Use `02-offer-architect.md` to choose the best format.
4. Use `03-workshop-designer.md` to prepare the room.
5. After the session, use `04-room-memory-follow-up.md` and `05-impact-dashboard.csv`.
6. Only use `06-content-repurposer.md` after the client has approved what can be shared.

## What Is Real Today

This starter kit is usable now as a document-based system. It includes templates, agent briefs, install notes, and a lightweight tracking CSV.

The next layer would be a private client workspace in Notion, Google Drive, Sheets, or a dedicated portal, with stronger automation around intake, follow-up, and reporting.

## Important Boundary

Keep client details private. Public pages can show the architecture, but client names, notes, recordings, internal tensions, and outcomes belong in controlled workspaces only.
