## What we built

The first cut of a registry that catalogs every agent operating across Frank's stack — what it can touch, what context it holds, and what it has shipped. Acts as the memory and IAM layer that Claude Code, Codex, and Antigravity all read from when they start a session.

## The stack and why

- **Claude Agent SDK** — primary runtime for orchestrator agents.
- **Postgres + MCP** — registry storage exposed via an MCP server so any tool can read it.
- **Next.js 16** — the public surface at `/starlight-intelligence-system`.

## What worked

- Modeling the registry as a flat capability list (not a hierarchy) made the IAM scoping trivial.

## What broke

- Cross-tool session resumption is still manual — Antigravity does not yet read the registry on start.

## One reusable artifact

Coming with the v0.1 ship.

## Next
