## What we built

A controlled comparison: implement the same product dashboard feature (filterable table with charts and CSV export) in Antigravity 2.0, Claude Code, and Codex. Track time, PR quality, test coverage, and the kinds of failures each tool produces. Full notes published as a benchmark report.

## The stack and why

All three tools, against the same feature spec, in the same repo, with the same acceptance criteria. The point is not "who wins" — the point is which tool earns which layer.

## What worked

- Antigravity dominated on greenfield UI plus browser validation.
- Claude Code dominated on the existing-repo PR work (refactors, test repair).
- Codex held its own on terminal-heavy tasks (CSV pipeline, data validation).

## What broke

- Cross-tool handoff is still rough. Each tool wants to own the full session; the orchestration layer between them does not exist yet (this is where SIS comes in).

## One reusable artifact

The benchmark report itself, published as a free PDF on `/build/template-pack`.

## Next

Repeat with a second feature (auth flow) to confirm the layer-ownership pattern is not feature-dependent.
