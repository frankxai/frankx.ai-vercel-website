## Day 1: Integration Gate

1. Resolve TypeScript and lint findings.
2. Add focused graph, import, preview-config, and route contract tests.
3. Run one production build under the bounded machine posture.
4. Create design evidence and validate it.
5. Run desktop, mobile, keyboard, and reduced-motion visual QA.
6. Ask a second provider for an adversarial review.
7. Batch one commit, push once, open a draft PR, and verify one Vercel preview.

Exit condition: preview is READY, routes render without runtime errors, and blockers are written into the PR.

## Day 2: Provider and Media Gate

1. Keep demo adapters as the default.
2. Define server adapter contracts for image, video, and 3D.
3. Benchmark one approved provider per modality against quality, latency, cost, rights, and failure handling.
4. Add moderation, spend ceiling, idempotency, job status, storage, and provenance requirements.
5. Produce a launch-film beat sheet before generating any paid video.
6. Require an inspectable GLB and poster fallback before claiming 3D support.

Exit condition: a provider decision record exists. No paid or production provider is activated without Frank.

## Day 3: Template Productization

Prioritize packages by reusable customer value:

1. Premium product launch system.
2. AI product and agent application shell.
3. Operational dashboard and command center.
4. Newsletter and editorial publication system.
5. Creator portfolio and world-building experience.
6. Marketplace and product-detail system.

Each package must include real content, target audience, information architecture, responsive states, data contracts, asset plan, accessibility, performance budget, tests, provenance, license, documentation, and a stable demo.

Exit condition: one additional package reaches preview quality; the rest remain scored briefs.

## Product Guardrails

- Do not copy Figma Weave trade dress, copy, media, or proprietary implementation.
- Use modern node-based creative products as capability research, then build an original FrankX system.
- Do not label community gallery templates as official-author templates.
- Do not put private v0 identifiers or tokenized preview URLs into public source.
- Do not represent a still as video, a poster as 3D, or a demo adapter as a production provider.
- Do not merge stacked Claude PRs wholesale.
- Do not create a second deployment path or a new Vercel project.
- Do not promote production without the explicit human gate.
