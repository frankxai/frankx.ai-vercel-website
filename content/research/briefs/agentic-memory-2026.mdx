# Agentic Memory — Research Brief

**TL;DR**: Context engineering manages the window. Agentic memory manages what survives the window across tools and sessions. 2026 leaders split into vector-first (Mem0), temporal graphs (Graphiti/Zep), and OS-tiered self-editing memory (Letta). Sovereignty-first fleets put `local_core` authority first and treat vendors as adapters.

## Taxonomy

| Type | Holds | Lifetime |
|------|-------|----------|
| Working / STM | Turn state, tool blobs | Minutes |
| Episodic | Runs, decisions, receipts | Days–years |
| Semantic | Facts, prefs, entities | Long |
| Procedural | Skills, playbooks | Versioned |
| Controller | Memory ops policy | Persistent |

---

## Architecture Bets

1. **Vector-first memory layers** — extract → embed → retrieve (Mem0-class)
2. **Temporal knowledge graphs** — time-stamped relations (Graphiti/Zep)
3. **OS-tiered memory** — context as RAM, store as disk (Letta/MemGPT lineage)
4. **Sovereign local core + adapters** — Starlight Memory doctrine

---

## Patterns

- Read-before-reason / write-after-act
- Memory that earns its spot (rules > dumps)
- Progressive disclosure of skills
- Actor-aware multi-agent provenance
- Compaction + structured notes outside transcript
- Export + dual-write policy (local canonical)

---

## Benchmarks

- **LongMemEval** — multi-session, temporal, abstention
- **LOCOMO** — long multi-session multi-hop dialogue
- **Private harness evals** — rehydrate new session; measure critical fact recall before action
- **Trajectory audits** — did retrieve/write happen at the right steps?
