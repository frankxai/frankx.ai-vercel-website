# Agentic Life Architecture — Research Brief

**TL;DR**: This is not a better chatbot memory plugin. Agentic life infrastructure is a multi-domain operating layer: memory that compounds across tools/sessions, skills that compose into modules, exportable ownership, trajectory receipts for swarm quality, and life domains (code, creator, wealth, brand, health, research) on one intelligence substrate.

---

## Problem Statement (Five Failure Modes)

| # | Failure mode | Symptom | Infrastructure response |
|---|--------------|---------|-------------------------|
| 1 | Context does not compound | Re-explain across tools/sessions | Memory contract + handoffs + vault |
| 2 | Specialization does not compose | Skills/agents don't form an OS | Contracts, progressive disclosure, team profiles |
| 3 | Sovereignty is fake | Lock-in, no export, silent data | Local-core authority + adapters |
| 4 | Quality is unverifiable | "Swarm worked" with no proof | Trajectory evals + receipts + maker≠checker |
| 5 | Life is multi-domain; tools are single-feature | Duct-taped stacks | ALOS-style modules + shared ledger |

## Countermeasures (What Helps)

1. **Compounding context** — write/select/compress/isolate; progressive skills; OS-tiered memory; temporal graphs; sovereign vault markdown.
2. **Composition** — agent/skill contracts; independent verifier; hierarchical skill routing before phase-transition collapse.
3. **Sovereignty** — exportable atoms; privacy classes; no silent training; adapters never own IDs.
4. **Quality** — trajectory eval > final answer; receipt JSON; offline regression + online tracing.
5. **Multi-domain life** — module map + daily command loop + fail-closed money/health paths.
