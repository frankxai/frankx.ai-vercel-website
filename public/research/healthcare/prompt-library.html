Diagnostic Similar Case Search
Tier 0 • Database

Pure SQL vector similarity search — no LLM tokens consumed. Returns most similar historical cases with outcomes.

-- No LLM needed: pure vector search SELECT p.patient_id, o.diagnosis_date, o.outcome_status, o.event_free_survival_months, ROUND(1 - VECTOR_DISTANCE( p.diagnosis_vec, (SELECT diagnosis_vec FROM patient_embeddings WHERE patient_id = {{PATIENT_ID}}), COSINE ), 3) AS similarity FROM patient_embeddings p JOIN treatment_outcomes o ON p.patient_id = o.patient_id WHERE p.patient_id != {{PATIENT_ID}} ORDER BY similarity DESC FETCH FIRST 10 ROWS ONLY;

Research Literature RAG Query
Tier 1 • PubMedBERT + Rerank

Retrieves and ranks relevant publications based on patient characteristics. Returns citations with relevance scores.

-- Hybrid search: keyword + semantic WITH patient_context AS ( SELECT diagnosis, mutations, risk_group FROM patient_summary WHERE patient_id = {{PATIENT_ID}} ) SELECT l.pmid, l.title, l.authors, l.journal, l.pub_year, RERANK_SCORE( l.abstract_embedding, p.diagnosis || ' ' || p.mutations, 'cohere-rerank-3.5' ) AS relevance FROM literature_embeddings l CROSS JOIN patient_context p WHERE VECTOR_DISTANCE( l.abstract_embedding, EMBED(p.diagnosis || ' ' || p.mutations), COSINE ) < 0.3 ORDER BY relevance DESC FETCH FIRST 20 ROWS ONLY;

Analysis Risk Factor Analysis
Tier 0 • Oracle ML

Explains XGBoost risk model predictions using SHAP values. No LLM needed — pure ML interpretability.

-- Model explanation via Oracle ML SELECT feature_name, feature_value, shap_value, CASE WHEN shap_value > 0 THEN 'Increases risk' ELSE 'Decreases risk' END AS impact_direction, ABS(shap_value) AS impact_magnitude FROM DBMS_DATA_MINING.EXPLAIN_MODEL( model_name => 'RELAPSE_RISK_XGBOOST', patient_id => {{PATIENT_ID}} ) ORDER BY impact_magnitude DESC FETCH FIRST 10 ROWS ONLY;
