# Jojo Hospitality Intelligence Kit

Version: 0.1.0  
Release date: 2026-06-25  
Public page: https://frankx.ai/downloads/jojo-hospitality-intelligence-kit

This is a starter system for Jojo, Harzfenster by Johannes Steingrueber, and Hotel Goertler. It is designed to help a serious owner-led restaurant and hotel carry new attention with calmer booking, better service preparation, better follow-up, team learning, and private house memory.

The kit is intentionally simple. Start with documents. Add Codex, Claude, ACOS, or automation only after one workflow proves useful.

## Start Here

1. Read `01-owner-operating-map.md`.
2. Pick one pilot workflow:
   - booking and pre-arrival
   - restaurant service briefing
   - hotel stay handoff
   - review and reputation learning
3. Read the matching file and copy the template into a private workspace.
4. Use the agents as drafting partners. A human approves anything that reaches a guest.
5. Review `privacy-boundaries.md` before putting any real guest or staff information into a system.

## What This Is

- A practical operating kit for hospitality work.
- A private-workspace starter pack.
- A bridge from a real friend/ally pilot into a reusable FrankX Hospitality Intelligence vertical.
- A way to make AI useful without making hospitality feel automated.

## What This Is Not

- Not a public CRM.
- Not a replacement for host judgment.
- Not a place for guest secrets, medical details, payment data, or private team records.
- Not a claim that any public result came from AI.
- Not a plugin yet.

## Suggested First Pilot

Run one workflow for 14 days:

- Daily service briefing from booking notes, menu priorities, and house reminders.
- One private debrief after service.
- One weekly owner dashboard with patterns, risks, and decisions.

If that helps, add booking reply drafts or review response drafts next.
