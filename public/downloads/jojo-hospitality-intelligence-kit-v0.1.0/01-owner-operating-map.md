# 01 - Owner Operating Map

Purpose: give Jojo one clear map for using AI as quiet operating support across Harzfenster and Hotel Goertler.

## North Star

The guest should feel more personally cared for, not more processed.

AI is useful when it improves preparation, memory, wording, and learning. It is wrong when it replaces human judgment, invents promises, or makes the house feel synthetic.

## House Loops

### 1. Attention Loop

Inputs:

- Michelin and press attention.
- Website visits.
- Reservation demand.
- Hotel inquiries.
- Messages, calls, and special requests.

Outputs:

- Clear public information.
- Faster but human-approved replies.
- Better waitlist handling.
- Better pre-arrival preparation.

### 2. Guest Loop

Inputs:

- Reservation details.
- Allergies and preferences.
- Occasion and timing.
- Room stay or local travel context.

Outputs:

- Service briefing.
- Host notes.
- Follow-up language.
- Repeat-guest memory if allowed and stored privately.

### 3. Team Loop

Inputs:

- Owner priorities.
- Menu changes.
- Producer notes.
- Known service risks.
- Feedback from the last services.

Outputs:

- Daily briefing.
- Staff learning notes.
- Training snippets.
- Weekly owner dashboard.

### 4. Story Loop

Inputs:

- Approved public facts.
- Menu themes.
- Producer stories.
- Local Harz context.
- Public review patterns.

Outputs:

- Website updates.
- Newsletter ideas.
- Social posts.
- Event descriptions.
- Workshop examples with no private guest data.

## First Operating Board

Create a private board with these sections:

- Today: services, hotel arrivals, risks, VIPs, restrictions.
- This week: events, menu changes, staff needs, review themes.
- House memory: standards, suppliers, phrases, common answers.
- Content approved: material safe to publish.
- Decisions: owner choices and unresolved questions.

## One Useful First Routine

Daily service briefing:

1. Pull booking notes.
2. Add menu and producer updates.
3. Add special guest context.
4. Add house reminders.
5. Draft briefing.
6. Human checks.
7. Team uses it.
8. After service, write a five-minute debrief.
