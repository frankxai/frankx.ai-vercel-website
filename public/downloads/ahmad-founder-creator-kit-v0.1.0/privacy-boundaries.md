# Privacy Boundaries

This kit is for private use first.

## Never Publish Without Approval

- Private family stories.
- Customer names or details.
- Screenshots from private systems.
- Local machine paths.
- Access tokens, API keys, passwords, or recovery codes.
- Client messages.
- Internal strategy notes.
- Financial details.
- Medical or health claims.
- Investment claims.

## Public-Safe Language

Use:

- "Public profiles connect Ahmad with..."
- "FrankX proposes..."
- "The starter kit helps..."
- "The first version is document-based..."
- "The local lab should stay private until approved..."

Avoid:

- "Ahmad already has a fully autonomous system."
- "The AI will run the business."
- "Guaranteed revenue."
- "Medical benefits."
- "Private proof without consent."

## Tooling Rule

ACOS, Codex, Claude, OpenClaw, Hermes-style routines, and local voice tools are optional power layers. They should not be presented as the product.

The product is the founder system Ahmad can use to decide, build, publish, learn, and improve.
