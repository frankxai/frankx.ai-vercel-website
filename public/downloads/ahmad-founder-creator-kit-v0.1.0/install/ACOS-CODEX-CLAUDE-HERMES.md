# Install Guidance - ACOS, Codex, Claude, Hermes-Style Routines

Use this order. Do not install everything first.

## Step 1 - Workspace

Create a private folder:

```text
ahmad-founder-creator-kit/
  README.md
  founder-notes/
  offers/
  content-library/
  proof/
  agents/
  repos/
```

## Step 2 - Use The Kit Manually

Before any automation:

1. Read the offer map.
2. Pick one weekly goal.
3. Use one agent brief manually in Codex or Claude.
4. Publish only after human approval.

## Step 3 - Add ACOS

Use FrankX ACOS downloads only when Ahmad wants Codex or Claude to help run repeatable workflows.

FrankX ACOS page:

https://frankx.ai/downloads/preview/agentic-creator-os

Recommended first ACOS use:

- Content strategist prompt.
- Founder chief of staff weekly plan.
- Brand guardian review.
- Repo accelerator checklist.

## Step 4 - Add Codex And Claude

Codex:

- repo work.
- type checks.
- page edits.
- packaging.
- local workflow scripts.

Claude:

- positioning.
- critique.
- long-form writing.
- workshop design.
- reflective synthesis.

Use both with the same privacy boundaries.

## Step 5 - Add OpenClaw Or Local Operator Layer

Only after:

- first offer is clear.
- workspace exists.
- manual workflow works.
- private data rules are explicit.

Start with read-only routines and human approval.

## Step 6 - Hermes-Style Routines

Hermes here means a named operator routine, not a public claim.

Good first routines:

- morning founder brief.
- content extraction.
- repo status.
- weekly proof review.
- next action card.

Bad first routines:

- autonomous posting.
- autonomous sales messages.
- autonomous file cleanup.
- credential access.
- money movement.
