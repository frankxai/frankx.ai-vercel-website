# 09 - Client Proof And Reflection Loop

Proof should be useful before it becomes public.

## Capture After Every Session

- What did the person ask for?
- What did they actually need?
- What language made sense to them?
- What confused them?
- What should Ahmad do differently next time?
- What artifact was created?

## Private Proof Types

- Session notes.
- Before and after operating map.
- Demo screenshots.
- Client question bank.
- Objection list.
- Tool checklist.
- Founder reflection.

## Public Proof Types

Only publish after approval:

- Anonymous pattern.
- Approved testimonial.
- Demo without private data.
- Short case note.
- Reusable template.
- Process diagram.

## Weekly Review

Every Friday:

1. Read the week's proof notes.
2. Pick one pattern.
3. Improve one offer sentence.
4. Improve one agent instruction.
5. Improve one content template.
6. Choose next week's experiment.
