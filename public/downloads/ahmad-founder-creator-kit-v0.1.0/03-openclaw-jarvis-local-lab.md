# 03 - OpenClaw Jarvis Local Lab

This is the private technical direction. Keep it off public pages until Ahmad approves screenshots, architecture, and claims.

## Goal

Create a local AI workstation that helps Ahmad:

- Capture ideas.
- Run Codex and Claude workflows.
- Coordinate OpenClaw-style local actions.
- Use voice for quick capture and control.
- Keep private memory private.
- Produce content faster without publishing raw drafts.

## Suggested Hardware

- Mac mini or Mac Studio.
- External drive for media and project archives.
- Good microphone for voice capture.
- Backup strategy before adding automation.

## Software Layers

1. Workspace: GitHub repos, local folders, notes, media library.
2. Agent tools: Codex, Claude, ACOS packs, and selected skills.
3. Local operator: OpenClaw or equivalent control layer.
4. Voice: Starlight Voice or another push-to-talk operator pattern.
5. Memory: SIS/SIP-inspired private vaults, not public training data.
6. Media: ArcaneaClaw-style classification, processing, captioning, and social prep.

## First Local Lab Routine

1. Press-to-talk capture: "What am I building today?"
2. Codex turns the answer into a task plan.
3. Claude critiques the plan for clarity and risk.
4. The founder chief of staff agent writes the day card.
5. The content strategist extracts one publishable idea.
6. Human approval before any public post.

## Safety Defaults

- No autonomous posting in v1.
- No credential access without explicit setup.
- No customer or family private data in public repos.
- No money movement.
- No medical or investment claims without expert review.
