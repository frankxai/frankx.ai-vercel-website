Findings: Design Excellence System

Figma: Is it Essential?

Verdict: No — but strategically valuable

Code-first is better for FrankX workflow:

  • lib/design/tokens.ts = design token source of truth (not Figma)
  • tailwind.config.js + globals.css = where tokens live
  • Our components are in code — Figma mirrors, doesn't lead
  • Apple HIG + M3 patterns already extracted to code (Feb 24, 2026)

Where Figma is uniquely useful:

  1. Visual composition before coding complex layouts
  2. Community reference files (Apple, shadcn, Material)
  3. Stakeholder design review (non-dev eyes)
  4. Code Connect: Figma component → correct React import
  5. get_design_context MCP: when Frank draws something new and wants Claude to implement it

Figma alternatives that work without Figma:

  • screenshots skill — browser screenshot → design review
  • v0.dev via MCP — prompt → component
  • nanobanana — image gen for visual concepts
  • playwright — visual regression testing

MCP Budget: 14 calls remaining (~15 total, 1 used)

Use for: High-value moments — Frank draws a layout, needs it coded. Don't use for: Routine component builds, token changes, content updates.
