Product Retro — Team Retrospective

Analyze what worked and what didn't in a product build cycle. Codify successful patterns into reusable knowledge.

Input

  • Product: $ARGUMENTS (product name or page path that was recently built/deployed)

Step 1: Gather Data

  1. Check git log for recent commits related to this product
  2. Read the product's page files (what was built)
  3. Check if QA issues were found and fixed
  4. Review any deployment issues

Step 2: Analyze Patterns

What Worked

  • Which components were reused effectively?
  • Which design patterns shipped cleanly?
  • What was the build → QA → deploy cycle time?
  • Which Task agents produced good results?

What Didn't Work

  • Where did we hit errors or need rework?
  • Which quality gates caught real issues?
  • What was missing from the spec?
  • Where did agents need human intervention?

Lessons Learned

  • New patterns to codify (add to skill.md files)
  • Anti-patterns to document (add to agent definitions)
  • Missing components that should exist
  • Process improvements for next sprint

Step 3: Update Knowledge Base

If clear patterns emerged:

  1. Update skill.md files with new best practices
  2. Update PRODUCT_TEAMS.md if workflow needs adjustment
  3. Log to memory if the insight is worth remembering across sessions

Iteration Mindset

  • Small improvements compound. Don't try to fix everything at once.
