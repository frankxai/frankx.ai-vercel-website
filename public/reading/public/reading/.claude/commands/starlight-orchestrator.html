/starlight-orchestrator — SIS Executive Layer

Purpose: The conductor of the Starlight Intelligence System. Routes tasks to appropriate agents/skills, coordinates parallel execution, synthesizes outputs, and manages n8n workflow infrastructure via MCP.

Input

Provide ONE of:

  • route [task] — analyze and route a complex task to the right agents
  • coordinate [swarm name] — activate and coordinate a named swarm
  • workflow [action] — manage n8n workflows (create/fix/status/activate)
  • audit — audit current system state (skills, workflows, agents, n8n health)
  • plan [project] — design full orchestration plan for a project
  • No input: show current system state + available orchestration options

Flow 4: Swarm Coordination

→ Route to Frank for approval (if outbound)

Process

Step 1: Classify the Request

ROUTE request?     → decompose + route to skills/agents
WORKFLOW request?  → use n8n-mcp tools
AUDIT request?     → system health check
PLAN request?      → design orchestration blueprint
No input?          → show system state

Step 2: Decompose Complex Tasks

For any complex request:

  1. List all skills/agents that could contribute
  2. Identify dependencies (what needs to run first)
  3. Identify parallelizable tasks (what can run simultaneously)
  4. Design execution order
  5. Set quality gates

Step 3: Execute

Invoke skills and tools in designed order. For parallel tasks: invoke simultaneously in one response. For sequential: wait for dependencies before proceeding.

Step 4: n8n Workflow Operations

Available via n8n MCP:

Action MCP Tool
List workflows n8n_list_workflows
Get workflow n8n_get_workflow
Create workflow n8n_create_workflow
Update workflow n8n_update_full_workflow
Test workflow n8n_test_workflow
Check health n8n_health_check
Get executions n8n_executions

Step 5: Validate and Synthesize

Before final output:

  • All agent outputs align with Brand DNA
  • No contradictions between outputs
  • Quality gates passed
  • Outbound content flagged for Frank's approval
