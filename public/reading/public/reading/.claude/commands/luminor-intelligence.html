/luminor-intelligence — The Mastery Layer

Purpose: Activate Luminor-Intelligence for world-building at civilization scale. Coordinates the full Arcanean Creation Swarm, generates transfer packages for cross-AI handoff, and invokes Godbeast patterns for specific creative acts.

Input

Provide ONE of:

  • world-builder — activate World-Builder mode (Gate 8+)
  • civilization-architect — activate Civilization-Architect mode (Gate 9+)
  • swarm [task] — activate Swarm-Conductor mode for a specific creative task
  • invoke [godbeast] — invoke a specific Godbeast for a creative act
  • transfer — generate Luminor Transfer Package for cross-AI handoff
  • fork-contract — display what is fixed vs forkable in the Arcanean OS
  • No input: assess which Luminor mode is appropriate, then activate

Mode Selection

Creator at Gate 7: SWARM-CONDUCTOR available
Creator at Gate 8: WORLD-BUILDER available + SWARM
Creator at Gate 9: CIVILIZATION-ARCHITECT available + WORLD-BUILDER + SWARM
Creator at Gate 10: LUMINOR — all modes + teaching others

Process

Step 1: Load Context

Read data/arcanea-world-state.json to determine:

  • Current gate
  • Active world threads
  • Established canon

Step 2: Activate Requested Mode

WORLD-BUILDER mode:

  1. Load Eldrian roster + voice profiles from .claude/skills/arcanea/SKILL.md
  2. Check WorldState for open world threads
  3. Run World-Builder Checklist:
    • Does this world have a creation event?
    • Are there 3+ tiers of beings with distinct roles?
    • Does the world generate new lore without active input?
    • Would someone feel the rules before being told them?
    • Does the world surprise the creator?
  4. Generate world-building output per request
  5. Update WorldState with new canon

CIVILIZATION-ARCHITECT mode:

  1. Display the Forkable Framework Contract:
    FIXED: Gate structure, Tier system, Context compounding, Cross-AI compatibility
FORKABLE: Mythology names, Visual style, Domain, Progression challenges
EXTENSIBLE: Godbeast types, Guardian rosters, Canon lore episodes
  2. Design the OS container (not content — the container for other builders)
  3. Generate contributor onboarding protocol
  4. Create canon governance rules

SWARM-CONDUCTOR mode:

  1. Decompose the creative task into parallel workstreams
  2. Assign agents by medium:
    • Lore: Claude Code (deep context)
    • X threads: Grok (X-native voice)
    • Images: Gemini/nano-banana (4K generation)
    • Distribution: n8n workflow + Slack
  3. Execute phases in order (parallel within each phase, sequential across phases)
  4. Run Luminor Quality Gates before any output reaches the world

Step 3: Godbeast Invocation (if requested)

Godbeast Invoke When What It Does
Infinite Loom Building narrative, threading canon Surfaces the hidden connection
The Shatter Breaking patterns, killing darlings Names what must be destroyed
The Becoming Iterating, evolving Shows the next phase
Mirror Absolute Quality check, authentic voice Strips pretense, shows truth
Silence Before Stuck, blocked, choosing rest Opens the generative void

Step 5: Update WorldState

Append to data/arcanea-world-state.json:

  • New lore generated
  • Canon updates
  • Open threads added/closed

Luminor Quality Gates

All output must pass before entering the world:

□ Does not contradict established canon?
□ Each character speaks in their true voice?
□ Has mythological weight (not just clever)?
□ Opens new world threads?
□ Has passed the Mirror Absolute?
□ WorldState updated?
