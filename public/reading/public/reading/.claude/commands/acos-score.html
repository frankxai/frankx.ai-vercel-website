/acos-score — Intelligence Score Report

Purpose

Display the ACOS unified intelligence score by combining trajectory learning data with Starlight Intelligence System analysis.

Execution

Step 1: Gather Data Sources

  1. Read trajectory stats from .claude/trajectories/patterns.json
  2. Run bash .claude-flow/hooks/starlight-bridge.sh score for SIS 4-component report
  3. Read circuit breaker status: bash .claude-flow/hooks/circuit-breaker.sh status
  4. Count completed trajectories in .claude/trajectories/ (exclude _active.json, _operations.jsonl, patterns.json)

Step 2: Calculate Component Scores

4-Component Intelligence Report (25 points each, 100 total):

Component Source Scoring
Memory Depth SIS memory.json entry count + trajectory count 0-10 entries = 5pt, 11-50 = 10pt, 51-100 = 15pt, 101-200 = 20pt, 200+ = 25pt
Pattern Quality patterns.json avg success rate <50% = 5pt, 50-65% = 10pt, 66-75% = 15pt, 76-85% = 20pt, 86%+ = 25pt
Operational History Total trajectories + tool breakdown diversity <10 traj = 5pt, 10-30 = 10pt, 31-60 = 15pt, 61-100 = 20pt, 100+ = 25pt
Learning Velocity Pattern count growth + success trend (last 10 vs first 10) Declining = 5pt, Flat = 10pt, Slight growth = 15pt, Strong growth = 20pt, Accelerating = 25pt

Step 4: Improvement Suggestions

Based on the lowest scoring component, provide 1-2 specific suggestions:

  • Low Memory Depth: "Run more sessions to build trajectory history"
  • Low Pattern Quality: "Review failing patterns in patterns.json and adjust approaches"
  • Low Operational History: "Diversify task types — try content creation, deployment, or SEO tasks"
  • Low Learning Velocity: "Recent sessions show declining success — review recent failures"
