/video-to-blog — Transform Video Transcript into Blog Post

Purpose

Convert a video transcript into a structured, SEO-optimized MDX blog post that follows FrankX content standards.

Usage

/video-to-blog {videoId|transcript-path} [--style tutorial|reaction|deep-dive] [--pillar ai|music|creator|investor|opinion]

Arguments

  • videoId — YouTube video ID (looks up transcript in data/video-transcripts/)
  • transcript-path — Direct path to a .txt or .srt transcript file
  • --style — Blog post style (default: tutorial)
    • tutorial — Step-by-step with code blocks and actionable takeaways
    • reaction — Opinion-led analysis with quotes and counterpoints
    • deep-dive — Long-form technical exploration with diagrams
  • --pillar — Content pillar for categorization and stream assignment

Pipeline

Step 1: Load Source Material

  1. Find transcript: check data/video-transcripts/{videoId}.txt or use direct path
  2. Load summary if available: data/video-transcripts/{videoId}-summary.md
  3. Load video metadata from data/video-vault-100.json (title, description, tags)
  4. Read docs/strategy/CONTENT_VOICE_DNA.md for voice rules

Step 2: Content Transformation

Transform the transcript into blog structure:

  1. Extract key sections from transcript timestamps/chapters
  2. Rewrite — NOT a transcript dump. Restructure for reading:
    • Remove filler words, verbal tics, repetition
    • Convert spoken examples into written code blocks or lists
    • Add context that was visual in the video (screenshots → descriptions)
    • Expand abbreviated explanations into full written form
  3. Add structure:
    • TL;DR in first 100 words
    • Question-based H2 headings
    • Code blocks with language tags where applicable
    • Callout boxes for key insights
    • Internal links to related FrankX content (3+ minimum)

Step 3: SEO Optimization

  1. Generate frontmatter:
    title: "{SEO-optimized title, 50-60 chars}"
description: "{Meta description, 150-160 chars}"
date: "{today YYYY-MM-DD}"
author: "Frank"
tags: ["{relevant}", "{tags}", "{from pillar}"]
image: "/images/blog/{slug}-hero.png"
videoSource: "{youtube URL}"
  2. Add FAQ section (5+ questions based on video content)
  3. Add schema markup hints in frontmatter for Article + FAQPage + VideoObject
  4. Generate 3 internal link suggestions from existing blog content

Step 4: Quality Gate

Before saving, verify:

  • TL;DR present in first 100 words
  • 3+ internal links included
  • Question-based H2s used
  • No transcript artifacts (timestamps, "um", "you know", speaker labels)
  • Code blocks have language tags
  • Image placeholder for hero image noted
  • Word count: 1,500-3,000 words (tutorial), 800-1,500 (reaction), 2,500-5,000 (deep-dive)
  • Voice matches CONTENT_VOICE_DNA.md rules (no emojis, no guru language, lead with results)

Step 5: Save and Cross-Reference

  1. Save to content/blog/{slug}.mdx
  2. Add entry to data/content-index.json via the content scanner
  3. Add cross-reference in data/video-cross-refs.json:
    {
  "videoId": "{id}",
  "blogSlug": "{slug}",
  "direction": "video-to-blog",
  "generatedAt": "{ISO timestamp}"
}
  4. If video exists in vault, add relatedBlog field to its entry
  5. Generate hero image prompt (Dark Premium Glass style) and note in output

Output

  • content/blog/{slug}.mdx — the blog post
  • Updated data/video-cross-refs.json
  • Hero image prompt for /generate-images or nano-banana

Style Templates

Tutorial (default)

TL;DR → Problem Statement → Step-by-Step Solution → Results → FAQ → Related

Reaction

TL;DR → The Claim → My Take → Evidence → What This Means → FAQ → Related

Deep Dive

TL;DR → Background → Technical Analysis → Architecture → Implementation → Benchmarks → Conclusion → FAQ → Related

Example

/video-to-blog dQw4w9WgXcQ --style tutorial --pillar ai

→ Loads transcript, transforms into SEO-optimized tutorial blog post, saves to content/blog/, cross-references video.
