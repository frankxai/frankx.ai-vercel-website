Trigger System Health Check

Run a comprehensive health check on the FrankX automation infrastructure.

What this does

Checks the health of:

  1. n8n Railway instance (API reachable?)
  2. Active workflows (how many, which ones?)
  3. Execution engine (actually running nodes?)
  4. Slack connectivity
  5. Key webhook endpoints

Execution

Run these checks in parallel:

1. n8n API Health

curl -s "https://primary-production-ff336.up.railway.app/api/v1/workflows?limit=50" \
  -H "X-N8N-API-KEY: $(cat ~/.secrets/.env.master | grep N8N_API_KEY | cut -d= -f2)"

2. Count active vs inactive workflows

Parse the response and count workflows by active status.

3. Test webhook endpoint

curl -s -X POST "https://primary-production-ff336.up.railway.app/webhook/cmd-router" \
  -H "Content-Type: application/json" \
  -d '{"command": "STATUS"}'

4. Post health report to Slack

Format findings as a health dashboard and post to #explorations (C09CQ1Y2TTK).

Include:

  • :green_circle: / :red_circle: status for each component
  • Active workflow count
  • Last known execution time
  • Recommendations if anything is degraded
