MCP Tool Budget Strategy

Tool Priority Order (highest ROI first)

  1. get_design_context — Frank designs something new and wants it coded

    • ROI: Very high — generates accurate code from actual Figma design
    • Use: Complex new layouts, hero sections, custom components

  2. get_variable_defs — Token sync after Figma changes

    • ROI: High — ensures tokens stay in sync
    • Use: After Frank imports new tokens in Tokens Studio

  3. get_screenshot — Quick visual reference without full code

    • ROI: Medium — saves context without burning full call budget
    • Use: "Show me what this looks like" quickly

  4. get_metadata — Structural overview of file/page

    • ROI: Low-medium — use to navigate unfamiliar files
    • Use: First time accessing a new community file

When NOT to Use MCP

  • Routine component builds (use existing patterns)
  • Token changes (update code directly, then sync Figma via Tokens Studio)
  • Content updates (not a design task)
  • Character image placement (images are already in /public)

Figma-to-Code Workflow

Step 1: Get Context

get_design_context(nodeId, fileKey, clientFrameworks="react,next.js", clientLanguages="typescript,css")

Step 2: Map to FrankX Tokens

Figma Color → FrankX Token
──────────────────────────────────────────────────
white/slate-900  → bg/void #0a0a0b
slate-700        → bg/elevated #1a1a1f
slate-100/white  → text/primary rgba(255,255,255,0.95)
slate-400        → text/secondary rgba(255,255,255,0.72)
glass fill       → bg-white/5 backdrop-blur-xl
border           → border-white/10
accent color     → brand-purple / brand-blue / brand-emerald etc.
shadow           → elevation-1 / elevation-2 / glass

Step 3: Apply State Layers

Any interactive element gets:

import { interactiveSurface, stateLayer, focusRing } from "@/lib/design/tokens";
// Add: interactiveSurface + stateLayer.hover + focusRing.default

Step 4: Apply Motion

import { m3Curves, motionDurations } from "@/lib/design/motion";
// Entering: emphasized curve, normal duration
// Exiting: emphasizedAccelerate curve, fast duration

Step 5: Check Code Connect

docs/design-system/code-connect.json
→ If component name matches, use the mapped import
→ If not, add new mapping after building

Token Sync Protocol

Code → Figma (when tokens change in code)

  1. Update globals.css and tailwind.config.js
  2. Update docs/design-system/figma-tokens.json to match
  3. Tell Frank: "Import updated figma-tokens.json via Tokens Studio → Sync"

Figma → Code (when tokens change in Figma)

  1. Call get_variable_defs(nodeId, fileKey) via MCP
  2. Compare returned variables to figma-tokens.json
  3. Update globals.css CSS variables
  4. Update tailwind.config.js extensions
  5. Commit: "chore(design): sync tokens from Figma"

Community File Activation Guide

When Frank has duplicated a community file:

  1. Open the file in Figma, copy the URL
  2. Share URL with Claude Code
  3. Claude extracts fileKey from URL
  4. Claude calls get_metadata to explore structure
  5. Claude calls get_variable_defs to extract tokens
  6. Claude maps to FrankX token system
  7. Results documented in docs/design-system/
