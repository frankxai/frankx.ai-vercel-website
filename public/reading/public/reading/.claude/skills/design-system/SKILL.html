Design System Enforcement v2.0

7-layer token architecture for FrankX. Every visual decision traces to a token. Source of truth: lib/design/tokens.ts, lib/design/motion.ts, app/globals.css

When This Skill Activates

  • Creating or modifying UI components
  • Adding new pages or sections
  • Working with colors, typography, spacing, or effects
  • Building Figma components or reviewing designs
  • Running /design-gods in any mode

7-Layer Token Architecture

Layer 1: Color Primitives

emerald-500, cyan-400, purple-600, amber-500, rose-500
NEVER use directly — always through semantic layer

Layer 2: Semantic Colors

import { contentHierarchy } from "@/lib/design/tokens";
// contentHierarchy.primary   = "text-white/95"
// contentHierarchy.secondary = "text-white/72"
// contentHierarchy.tertiary  = "text-white/48"
// contentHierarchy.muted     = "text-white/32"

Layer 3: Surfaces

import { surface } from "@/lib/design/tokens";
// surface.glass       → backdrop-blur-xl bg-white/5 border border-white/10
// surface.elevated    → backdrop-blur-xl bg-white/[0.08] border border-white/15
// surface.subtle      → bg-white/[0.02] border border-white/5
// surface.transparent → bg-transparent border border-white/10

Layer 4: Elevation

import { elevation } from "@/lib/design/tokens";
// elevation.sm     → cards at rest (subtle)
// elevation.md     → hover, popovers (medium)
// elevation.lg     → modals, overlays (prominent)
// elevation.xl     → toasts (max)
// elevation.brand  → featured items (emerald glow)
// elevation.emerald/purple/cyan/gold → brand-tinted variants

Layer 5: Typography

font-display  → Poppins (headings ≥18px ONLY)
font-sans     → Inter (everything else)
font-mono     → JetBrains Mono (code ONLY)
font-serif-italic → Playfair Display (quotes ONLY)

Fluid type (preferred):
  text-fluid-xs through text-fluid-7xl (clamp() based)
  BETTER than: text-4xl lg:text-5xl (breakpoint overrides)

Line height:  lineHeight.tight/snug/normal/relaxed/headline/body
Letter spacing: letterSpacing.tight/normal/wide/wider/eyebrow

Layer 6: Spacing

import { sectionSpacing, componentPadding } from "@/lib/design/tokens";
// 4px base grid
// sectionSpacing.sm/md/lg/hero
// componentPadding.xs/sm/md/lg/xl

Section: py-24 lg:py-32 (standard), py-12 lg:py-16 (compact)
Hero:    pt-16 md:pt-20 pb-24 lg:pb-32
Content: max-w-7xl mx-auto px-4 sm:px-6 lg:px-8

Hard Rules

Colors

  • NEVER hardcode hex — use CSS variables or Tailwind tokens
  • NEVER use raw opacity for text — use contentHierarchy tokens
  • Background: #0a0a0b (void) base. Everything layered on top.
  • Glass opacity: 0.02–0.08 MAX. Never above 0.08.

Typography

  • Display font: headings ≥18px ONLY
  • Body font: everything else
  • Prefer fluid type (text-fluid-*) over breakpoint overrides
  • Stats/prices: add font-nums class for tabular numerics

Border Radius

  • Cards: rounded-4xl (32px)
  • Buttons: rounded-2xl (16px)
  • Inputs: rounded-lg (8px)
  • Pills/badges: rounded-full

Icons

  • Lucide React — import individually, NEVER import *
  • Sizes: 16 (inline) / 20 (buttons) / 24 (cards) / 32 (decorative)
  • Always strokeWidth={1.5}

Interactive Surfaces (M3 + Apple HIG)

import {
  interactiveSurface,
  stateLayer,
  focusRing,
  touchTarget,
} from "@/lib/design/tokens";
// ALL clickable elements need:
// interactiveSurface → relative overflow-hidden cursor-pointer select-none
// stateLayer.hover   → M3 8% white overlay (::before pseudo)
// focusRing.default  → ring-2 ring-emerald-400/60 (WCAG 2.2)
// touchTarget.standard → min-h-[44px] (Apple HIG)

Motion

  • M3 curves (NOT css ease-in-out)
  • prefers-reduced-motion: ALWAYS check
  • Entering: emphasized curve + normal duration
  • Exiting: emphasizedAccelerate + fast duration
  • Interaction: spring physics (motionSpring.interactive)

CSS Utilities (globals.css)

.glass-emerald  → tinted glass with emerald glow
.glass-purple   → tinted glass with purple glow
.glass-vibrancy → Apple-style saturate(1.8) backdrop
.surface-1..5   → stepped surface brightness
.font-nums      → tabular-nums for stats/prices
.text-fluid-*   → responsive clamp() typography
.transition-fast/normal/slow → motion duration utilities
.scrollbar-thin → styled scrollbar (webkit + firefox)

Anti-Patterns

Bad Good
bg-white/15 (too bright) bg-white/5 or surface.glass
text-gray-400 text-white/48 or contentHierarchy.tertiary
shadow-lg elevation.md or shadow-elevation-2
rounded-lg on cards rounded-4xl
transition-all duration-300 transition-colors + motionDurations.fast
font-poppins on body font-sans
ease-in-out m3Curves.emphasized
text-4xl lg:text-5xl text-fluid-4xl
Inline color="#10b981" className="text-emerald-500"
animate-spin without reduced-motion Check useReducedMotion() first
hardcoded bg-gradient-to-r from-* gradients.text.brand or gradientClasses

Figma-to-Code Mapping

When receiving Figma designs via MCP:

  1. Map Figma colors → CSS variables (never raw hex)
  2. Map Figma text styles → type scale tokens
  3. Map Figma effects → surface/elevation system
  4. Map Figma shadows → elevation.sm/md/lg
  5. Check component exists in components/ui/ before building new
  6. Apply interactiveSurface + stateLayer + focusRing on clickable elements
  7. Apply m3Curves for enter/exit motion
  8. Use lib/design/gradients.ts for gradients
