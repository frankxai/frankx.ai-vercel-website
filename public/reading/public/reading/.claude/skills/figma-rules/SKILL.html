When NOT to Use MCP

  • Routine component builds (use existing patterns instead)
  • Token changes that start in code (update code → sync Figma later)
  • Content updates (not a design task)
  • Character image placement (images already in /public)

Token Sync Protocol

Code → Figma (when tokens change in code)

  1. Update globals.css and tailwind.config.js
  2. Update docs/design-system/figma-tokens.json to match
  3. Tell Frank: "Import updated figma-tokens.json via Tokens Studio → Sync"

Figma → Code (when tokens change in Figma)

  1. Call get_variable_defs(nodeId, fileKey) via MCP
  2. Compare to docs/design-system/figma-tokens.json
  3. Update globals.css CSS variables + tailwind.config.js
