// Track successful workflows as trajectories
const jj = new JjWrapper();

// ACOS workflow tracking
jj.startTrajectory("Create blog article with SEO");
await executeACOSCommand("/frankx-ai-blog");
jj.addToTrajectory();
jj.finalizeTrajectory(0.9, "Article published, good engagement");

// Arcanea gate progression tracking
jj.startTrajectory("Complete Gate 5 Voice certification");
await completeGateExercises("voice");
jj.addToTrajectory();
jj.finalizeTrajectory(0.85, "Gate opened, need practice on social content");

// Get AI suggestions based on learned patterns

console.log(`Confidence: ${suggestion.confidence}`);

Architecture Decision Records

ADR-001: Profile-Based Skill Loading

Decision: Use JSON profiles to define skill sets per workflow Rationale: 630 skills is valuable but loading all wastes context Consequence: 5-10x more relevant skills per task

ADR-002: Arcanea Gate Skill Mapping

Decision: Map each Arcanea gate to specific skill clusters Rationale: Progressive skill unlocking matches progression system Consequence: Guided skill development pathway

ADR-003: Agentic Jujutsu for Learning

Decision: Use trajectory tracking for workflow optimization Rationale: Self-learning improves skill activation over time Consequence: System gets smarter with use

ADR-004: Claude Flow Absorption into ACOS

Decision: Port claude-flow V3 infrastructure directly into ACOS Rationale: Unified system > separate tools. Full hook pipeline, commands, agents, swarm coordination all in one place. Consequence: ACOS now has enterprise-grade orchestration: 7 hook events, matcher-based tool handling, swarm communication, checkpoint management, failure learning, pre-compact context preservation. Same architecture as Arcanea/oracle-work deployments.

ADR-005: Ultraworld - Arcanean Swarm World-Building

Decision: Create /ultraworld as the master creative orchestration command for Arcanea universe building. Rationale: World-building requires coordinated creative agents (World Architect, Lore Keeper, Character Forger, Story Weaver, Game Designer, Art Director, Voice Alchemist) working in concert. Manual coordination doesn't scale. Consequence: Single command auto-routes to the right creative agents based on intent. Hierarchical anti-drift topology ensures canon consistency. Self-learning trajectories make the creative swarm better over time. Deployed to both FrankX and Arcanea repos.

ADR-006: ACOS v7.0 Unified Auto-Router

Decision: Make /acos the single entry point that auto-routes ALL requests. Claude-flow commands become internal infrastructure. Rationale: 83+ claude-flow commands are overwhelming. Users should describe intent, not memorize commands. Auto-routing via keyword detection + hook system handles dispatch. Consequence: Users only need /acos, /ultraworld, and a handful of primary commands. Everything else is handled automatically. claude-flow-* commands preserved for backwards compatibility but never surfaced to users.
