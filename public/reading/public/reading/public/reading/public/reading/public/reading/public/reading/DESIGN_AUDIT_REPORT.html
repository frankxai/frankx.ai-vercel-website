FrankX Design System Audit Report

Comprehensive Analysis & Recommendations

Audit Date: 2026-01-14 Auditor: UI/UX Design Expert (Claude Agent) Scope: All components, pages, and design tokens

Executive Summary

The FrankX design system demonstrates strong visual aesthetics and emerging consistency, with a premium glassmorphic style that aligns with the brand's consciousness-first mission. However, there are 3 critical inconsistencies that need immediate attention to achieve world-class design system maturity.

Overall Design System Score: 7.2/10

Strengths:

  • Beautiful glassmorphic aesthetic with depth and premium feel
  • Strong color palette aligned with brand identity
  • Comprehensive Tailwind configuration with custom tokens
  • Accessibility-conscious components (min-height, focus rings)
  • Thoughtful animation and hover states

Critical Issues:

  • Button border-radius inconsistency (6px vs 16px)
  • No centralized gradient system (hardcoded throughout)
  • Brand color hex codes not mapped to CSS variables
  • Missing typography implementation (Poppins not loaded)

Detailed Scoring (WCAG 2.2 + Design Excellence Framework)

1. Visual Hierarchy & Layout: 8.5/10

Strengths

  • Clear focal points with gradient-enhanced headings
  • Excellent use of whitespace and glassmorphic depth
  • Consistent 8px spacing grid applied throughout
  • Beautiful radial gradient backgrounds for depth

Issues

  • Minor: Some sections use p-7 (28px) breaking 8px grid
  • Minor: Heading hierarchy occasionally skips levels (h1 → h3)

Recommendations

  1. Audit all padding/margin values for 8px grid compliance
  2. Enforce semantic heading hierarchy in all pages
  3. Create spacing utility classes for common patterns

Code Example - Fix Non-Grid Spacing:

// ❌ WRONG
<div className="p-7">  // 28px - breaks 8px grid

// ✅ CORRECT
<div className="p-6">  // 24px (3× base)
<div className="p-8">  // 32px (4× base)

2. Typography & Readability: 6.5/10

Strengths

  • Well-defined typography scale in Tailwind config
  • Proper line-height ratios (1.5× for body, 1.2× for headings)
  • Good contrast ratios (white on dark backgrounds)
  • Readable font sizes (16px minimum for body)

Critical Issues

  • CRITICAL: Poppins font (brand standard) not implemented
    • BRAND_IDENTITY.md specifies: --font-heading: 'Poppins'
    • tailwind.config.js uses: Inter for all text
  • Minor: Inconsistent font weight application (some headings use medium instead of bold)

Recommendations

  1. IMMEDIATE: Add Poppins to Next.js font configuration
  2. Map heading classes to use Poppins, body to use Inter
  3. Create typography component library for consistency
