Skill Development Guidelines

When to Create a New Skill

✅ Create a skill when:

  • You need specialized knowledge repeatedly (3+ times)
  • Knowledge is comprehensive (500+ words)
  • Applies across multiple contexts/projects
  • You want consistent approach to a domain

❌ Don't create a skill for:

  • One-time tasks (use slash command instead)
  • Simple facts or quick references
  • Personal preferences without methodology
  • Tasks better suited for MCP servers (data access)

Skill Template

When creating new skills, use this structure:

---
name: Skill Display Name
description: One-line description of what this skill does
version: 1.0.0
---

Skill Categories Explained

Technical vs Business vs Creative

Technical Skills:

  • Focus: How to build, architect, implement
  • Audience: Developers, engineers, architects
  • Output: Code, systems, technical decisions

Business Skills:

  • Focus: Strategy, planning, enterprise needs
  • Audience: Product managers, consultants, executives
  • Output: Strategies, roadmaps, presentations

Creative Skills:

  • Focus: Content, brand, expression, artistry
  • Audience: Creators, artists, marketers
  • Output: Content, designs, music, copy

Personal Skills:

  • Focus: Self-development, mindset, health
  • Audience: Yourself (Frank)
  • Output: Habits, decisions, personal growth

Project Skills:

  • Focus: Specific project context and workflows
  • Audience: Project team members
  • Output: Project-specific deliverables
