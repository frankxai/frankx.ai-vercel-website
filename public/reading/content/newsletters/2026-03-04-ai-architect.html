FROM THE ARCHITECT'S DESK

This week marks a shift I've been tracking for months: the best AI systems
aren't single models anymore -- they're engineered systems of specialist models
collaborating inside structured architectures. DeepSeek V4 activates only 32B
of its 1T parameters per token. Apple ships agentic coding directly into Xcode.
Claude Code now writes 4% of all GitHub commits. The era of "pick the best
model" is giving way to "design the best system." That's what this newsletter
is about.

1. DEEPSEEK V4: 1 TRILLION PARAMETERS, 32 BILLION ACTIVE
   Model Release | DeepSeek

DeepSeek V4 launched around March 3. The numbers: 1 trillion total parameters
with only 32 billion active per token -- fewer active parameters than V3 despite
being vastly larger. The model is fully multimodal (text, image, video).

Why This Matters: This is Mixture-of-Experts at serious scale. The key insight
isn't the parameter count -- it's that routing efficiency has improved to the
point where 97% of the model stays dormant on any given token. If you're
designing agentic systems, sparse activation means you can run frontier-quality
reasoning at mid-tier compute costs.

What You Can Do:
- Benchmark V4 against your current routing layer
- Test multimodal capabilities for document understanding pipelines
- Watch for the fine-grained sparse attention paper (50% efficiency gains)

DEEP DIVE: AGENTIC ENGINEERING -- WHY INTELLIGENCE BECAME A SYSTEM

The biggest architectural shift of 2026 isn't a single model release. It's the
recognition that production AI is an engineered system of specialists, not a
monolithic model.

The pattern:

  Planning Model      -> reasoning, strategy
  Extraction Model    -> parsing, structure
  Compliance Model    -> safety, policy
  Perception Model    -> vision, audio
  Generalist LLM      -> fallback, escalation
  Connected via MCP   -> universal tool interface

Small, specialist models collaborate inside structured agentic architectures.
Each model is chosen for its strengths, not forced to be good at everything.

Three converging forces made this pattern viable:

1. Sparse MoE at scale. DeepSeek V4 proves you can get frontier quality at
   mid-tier compute. Running multiple specialist models is now economical.

2. MCP as the universal tool interface. With all three major clouds shipping
   managed MCP servers, agents connect to any tool through a standard protocol.

3. Inference-time scaling beats training-time scaling. Most benchmark progress
   in 2026 comes from improved tooling and inference-time compute.

What to build toward: Stop optimizing for "which single model is best." Start
designing the orchestration layer -- the routing logic that sends each subtask
to the right specialist. That's where the competitive advantage lives in 2026.
