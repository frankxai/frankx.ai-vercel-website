ACOS Agent Standards

What Makes a Valid Agent

An agent must meet ALL criteria:

1. Clear Domain Ownership

  • Each agent owns a distinct domain (no overlap)
  • Domain boundaries are defined by keyword triggers
  • An agent's skills are specific to its domain

2. Skill Coverage

  • Must have at least 1 skill assigned
  • Skills must be from the approved skill roster (skill-rules.json)
  • Skills are loaded lazily — only when the agent is routed to

3. Routing Integration

  • Must have a corresponding route in guidance-hooks.sh
  • Route outputs actionable instructions (executable routing, not just labels)
  • Route includes: MCP tools, active skills, quality gates

4. Measurable Performance

  • Agent performance tracked via trajectory domain success rates
  • Agents with <30% success rate across 5+ sessions are flagged for review
  • Skill weights feed back into agent routing recommendations

Agent Lifecycle

Proposed → Guardian Review → Active → Performance Review → Permanent or Merged
  • New agents need 2/3 Guardian approval
  • Agents are reviewed every 30 days against trajectory data
  • Underperforming agents are merged into existing ones (not deleted)
  • Meta agents require 3/3 approval to modify

Addition Criteria

A new agent is justified when:

  • An existing agent's domain has grown too broad (>6 skills)
  • Trajectory data shows a clear domain cluster with unique tool patterns
  • The skill-builder identifies a domain gap not covered by existing agents
  • A new MCP server creates a distinct capability domain

Removal/Merge Criteria

An agent is merged when:

  • Its domain overlaps >70% with another agent
  • It handles fewer than 5 sessions per 30 days
  • Its success rate is consistently lower than the overlapping agent
