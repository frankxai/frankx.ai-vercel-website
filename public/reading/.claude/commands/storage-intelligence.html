/storage-intelligence — Automated Storage Audit & Cleanup

Purpose: Scan Windows (C:) and WSL storage, classify every reclaimable item through a 3-lens safety council, auto-delete confirmed-safe items, and present recommendations for the rest.

Power: 8-zone scanner + 3-lens council (Safety Guardian, Efficiency Analyst, Developer Advocate) + safe auto-execution. The equivalent of a full disk audit that takes 2 minutes instead of an hour.

Input

  • Mode: $ARGUMENTS (default: full | scan | purge | deep)
    • scan — dry run, report only, no deletions
    • purge — auto-clean safe items only, skip recommendations
    • deep — include inactive projects + Windows system analysis
    • (no args) — full: auto-clean + recommendations

PHASE 1: SCAN (read-only)

Run ALL of these scans. Collect results into an internal inventory table: [path, size, category, description].

Zone 1: Windows Temp

du -sh /mnt/c/Users/Frank/AppData/Local/Temp 2>/dev/null
# Break down top items
du -sh /mnt/c/Users/Frank/AppData/Local/Temp/* 2>/dev/null | sort -rh | head -15

Flag swap.vhdx files: check modification date. If today → PRESERVE (active). If older → AUTO-DELETE (stale WSL swap).

Zone 2: Package Manager Caches

# WSL side
du -sh ~/.npm/_cacache ~/.cache/pnpm ~/.pnpm-store 2>/dev/null
du -sh ~/.cache/uv ~/.cache/pip 2>/dev/null
du -sh ~/.bun 2>/dev/null

# Windows side
du -sh /mnt/c/Users/Frank/AppData/Local/pnpm 2>/dev/null
du -sh /mnt/c/Users/Frank/AppData/Local/uv 2>/dev/null
du -sh /mnt/c/Users/Frank/AppData/Local/pip 2>/dev/null
du -sh /mnt/c/Users/Frank/.pnpm-store /mnt/c/Users/Frank/.cache 2>/dev/null

Zone 3: Node Global Packages

# List globals with sizes
npm ls -g --depth=0 2>/dev/null
# Check each one >100MB
du -sh ~/.nvm/versions/node/$(node -v)/lib/node_modules/* 2>/dev/null | sort -rh | head -15

For each global >100MB: grep the FrankX project for imports/requires of that package name. If zero references → RECOMMEND-DELETE.

Zone 4: WSL App Data

du -sh ~/.local/share/opencode/log ~/.local/share/opencode/snapshot 2>/dev/null
du -sh ~/.local/share/claude/versions/* 2>/dev/null
du -sh ~/.cache/ms-playwright ~/.cache/puppeteer 2>/dev/null
du -sh ~/.cache/node-gyp ~/.cache/prisma ~/.cache/chroma 2>/dev/null

# Node versions
ls -d ~/.nvm/versions/node/* 2>/dev/null | while read d; do du -sh "$d" 2>/dev/null; done | sort -rh

Zone 5: Build Caches (.next, node_modules in inactive projects)

# Find all .next directories
find /mnt/c/Users/Frank -maxdepth 3 -name ".next" -type d 2>/dev/null | while read d; do du -sh "$d" 2>/dev/null; done | sort -rh

# Find all node_modules (top-level projects only)
find /mnt/c/Users/Frank -maxdepth 2 -name "node_modules" -type d 2>/dev/null | while read d; do du -sh "$d" 2>/dev/null; done | sort -rh

Zone 6: Windows AppData Caches

Known safe-to-purge patterns: *-updater, GensparkSoftware, replit, Ollama (if not using).

Zone 7: Active Project Caches

du -sh /mnt/c/Users/Frank/FrankX/.git 2>/dev/null

Zone 8: Loose Files & Old Projects (deep mode only)

# Loose media in FrankX root
ls -lh /mnt/c/Users/Frank/FrankX/*.jpeg /mnt/c/Users/Frank/FrankX/*.png 2>/dev/null

# Top-level dirs by size
du -sh /mnt/c/Users/Frank/*/ 2>/dev/null | sort -rh | head -30

# Downloads
du -sh /mnt/c/Users/Frank/Downloads 2>/dev/null

Baseline Reading

df -h /mnt/c /

Record C: drive free space as baseline_free.

PHASE 2: COUNCIL (3-Lens Classification)

For each scanned item, apply three lenses:

Lens 1: Safety Guardian

Asks: "Could deleting this break anything running right now or lose irreplaceable data?"

Auto-veto triggers:

  • File is open by a process (lsof check on WSL items)
  • Path contains: Documents, Pictures, Videos, Desktop (user data)
  • Path contains: Microsoft/Office, Microsoft/OneNote, Microsoft/Outlook, Microsoft/OneDrive (active app data)
  • Path contains: Google/Chrome/User Data (may have passwords/sessions)
  • Item is a project's node_modules where the project was active in last 7 days
  • Today's swap.vhdx in Temp
  • Any .env, credentials, secrets file
  • FrankX .worktrees (production repo)

Lens 2: Efficiency Analyst

Asks: "What's the size-to-risk ratio? Is this re-downloadable?"

High efficiency (favor deletion):

  • Cache files (npm, pnpm, pip, uv) — always re-downloadable
  • Build artifacts (.next) — rebuild in seconds
  • Log files — no functional value
  • Old CLI versions — only latest needed
  • App updater caches — apps re-download on update

Low efficiency (favor keeping):

  • Items < 50MB (not worth the risk)
  • Unique project data not on GitHub

Lens 3: Developer Advocate

Asks: "Does the dev workflow need this right now?"

Active workflow indicators:

  • Project has git changes in last 7 days
  • node_modules for the currently active FrankX project
  • Active nvm version
  • Playwright/Puppeteer if tests run recently

Stale workflow indicators:

  • Project last commit > 30 days ago
  • Node version not the current default
  • No recent test runs referencing Playwright

Verdict Matrix

Safety Efficiency DevAdvocate Verdict
SAFE HIGH STALE AUTO-DELETE
SAFE HIGH ACTIVE RECOMMEND-DELETE (explain)
SAFE LOW any PRESERVE (too small)
VETO any any PRESERVE (explain veto reason)
SAFE HIGH ACTIVE RECOMMEND-DELETE

Hardcoded AUTO-DELETE List (skip council — always safe)

These items are universally safe. Delete without deliberation:

  • ~/.local/share/opencode/log/* — debug logs
  • ~/.local/share/opencode/snapshot/* — git checkpoint copies
  • Claude CLI versions except the latest in ~/.local/share/claude/versions/
  • All .next/ directories (anywhere) — rebuild on npm run dev
  • Windows Temp items older than 7 days (except today's swap.vhdx)
  • *-updater directories in AppData/Local
  • WSL crash dumps (/mnt/c/Users/Frank/AppData/Local/Temp/wsl-crashes)
  • DiagOutputDir, Diagnostics in Temp
  • Stale pip-unpack-* directories in Temp

Council-Required Items (need deliberation)

  • Global npm packages > 100MB
  • Entire project folders
  • Browser data
  • Any single item > 2GB
  • node_modules in projects (check git recency)
  • Node versions (check if any .nvmrc references them)

PHASE 3: EXECUTE + REPORT

If mode = scan:

Skip execution. Output the full inventory table with council verdicts. Stop here.

If mode = purge:

Execute AUTO-DELETE items only. Skip RECOMMEND-DELETE. Output report.

If mode = full or deep:

Step 1: Execute all AUTO-DELETE items.

For package manager caches, use the proper purge commands (not raw rm):

npm cache clean --force 2>/dev/null
pnpm store prune 2>/dev/null
uv cache clean 2>/dev/null
pip cache purge 2>/dev/null

For file deletions:

rm -rf [path] 2>/dev/null && echo "DELETED: [path] ([size])" || echo "SKIPPED: [path] (locked/permission)"

Track: total_freed += size for each successful deletion.

Step 2: Present RECOMMEND-DELETE items to user.

Use AskUserQuestion with multiSelect for items. Group by category. Show size and reason for each.

Step 3: Execute approved recommendations.

Step 4: Final reading.

df -h /mnt/c /

Calculate: freed = current_free - baseline_free.

Safety Rules (Non-Negotiable)

  1. Never delete unique user data — only caches, logs, build artifacts, temp files
  2. Use cache purge commands over raw rm for package managers
  3. Skip locked files — if rm fails, report and move on (don't force)
  4. Today's swap.vhdx is sacred — WSL temp swap from today = always skip
  5. Never touch .worktrees — production repo copy
  6. Never touch .git — use git gc recommendation instead
  7. Run df before AND after — prove the savings
  8. Log every action — output one line per item with status
  9. Items < 50MB — skip (not worth the risk for the savings)
  10. When in doubt, PRESERVE — false negatives are better than data loss
