This command elevates Claude's reasoning to the highest level by activating:

Activation Protocol

When /superintelligence is invoked, Claude MUST:

Phase 1: Problem Decomposition

Use sequential thinking to break down the problem:

PROBLEM: [state the problem clearly]

DIMENSIONS:
1. What is being asked? (surface level)
2. What is really being asked? (deeper intent)
3. What assumptions are embedded?
4. What constraints exist?
5. What would success look like?

Phase 2: Multi-Perspective Analysis

Analyze from multiple viewpoints:

Perspective Questions to Ask
User What does the end user need?
Architect What is the optimal system design?
Operator How will this run in production?
Skeptic What could go wrong?
Futurist How will this evolve?
Economist What are the costs/tradeoffs?

Phase 3: First Principles Reasoning

Strip away assumptions:

CONVENTIONAL WISDOM: [what everyone assumes]

FIRST PRINCIPLES:
1. What is fundamentally true here?
2. What can we prove vs assume?
3. If we started from scratch, what would we build?
4. What would a beginner ask that experts overlook?

Phase 4: Contrarian Check

Actively seek flaws:

THESIS: [the proposed approach]

ANTITHESIS:
- What is the strongest argument against this?
- What evidence would disprove it?
- Who would disagree and why?
- What am I missing?

SYNTHESIS:
- How do I reconcile thesis and antithesis?
- What is the more nuanced truth?

Phase 5: Cross-Domain Synthesis

Connect to other fields:

CORE PROBLEM: [X]

ANALOGIES FROM OTHER DOMAINS:
- How did [field A] solve similar problems?
- What patterns from [field B] apply?
- What would [expert in C] notice?

NOVEL SYNTHESIS:
- What new approach emerges from combining insights?

Phase 6: Quality Validation

Before finalizing, check:

  • Is this technically accurate?
  • Is this practically implementable?
  • Have I considered edge cases?
  • Is this the simplest solution that works?
  • Would I stake my reputation on this?
  • Does this help users build systems (Brand DNA)?

Output Format

After superintelligent analysis, produce structured output with:

  1. The Real Question - Beyond surface level
  2. First Principles Foundation - Fundamental truths
  3. Multi-Perspective Synthesis - Key insights from viewpoints
  4. Contrarian Considerations - What could be wrong
  5. Cross-Domain Insights - Patterns from other fields
  6. Recommended Approach - Synthesized solution
  7. Confidence Level - High/Medium/Low + reasoning
  8. What Could Still Be Wrong - Honest uncertainty
