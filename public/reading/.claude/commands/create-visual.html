/create-visual — Autonomous Visual Creation Pipeline

Input: $ARGUMENTS (description of what image to create, e.g. "Golden Age hero", "Arcanea classroom replacement", "team assembly group shot")

Pipeline Overview

This command runs the full 6-step Visual Creation workflow autonomously. Every image passes through the council before generation. Every generation is logged. Nothing ships without human approval.

STEP 1: STUDY

  1. Read docs/strategy/IMAGE_STYLE_DNA.md (focus on organic-first principle + the 3 approved styles)
  2. Read data/visual-prompt-templates.json (load the matching template type)
  3. View the gold standard: Read public/images/mascot/mascot-v05-techno-beast-standing.png to recalibrate visual quality bar
  4. Determine:
    • Which of the 3 styles fits: Dark Premium Glass / Da Vinci Codex / Light Workshop
    • Which sub-variant if Dark Premium Glass: Cyberpunk Neon / Da Vinci Dark / Cinematic Studio / Holographic Command
    • What physical organizing metaphor anchors the composition (NOT "a futuristic visualization" — a real place/object)
    • 4+ named components with CAPS LABELS, hex colors, and materials

Self-check: Can you name the style, sub-variant, metaphor, and 4+ components? If not, study more.

STEP 2: DRAFT

Fill the structured template completely. ALL fields mandatory:

SUBJECT: [specific topic]
ORGANIZING METAPHOR: [physical scene/object — workbench, observatory, forest canopy, crystal cave]
STYLE: [approved style + sub-variant]
VIEWPOINT: [camera angle with degrees]

COMPONENTS (min 4):
1. NAME: [CAPS]  |  POSITION: [where]  |  COLOR: [name + #hex]  |  MATERIAL: [tactile]
2. NAME: [CAPS]  |  POSITION: [where]  |  COLOR: [name + #hex]  |  MATERIAL: [tactile]
3. NAME: [CAPS]  |  POSITION: [where]  |  COLOR: [name + #hex]  |  MATERIAL: [tactile]
4. NAME: [CAPS]  |  POSITION: [where]  |  COLOR: [name + #hex]  |  MATERIAL: [tactile]

ORGANIC ANCHOR: [living/organic element — creature, root network, bark, crystal growth, moss]
FOREGROUND: [specific elements]
MIDGROUND: [primary subject]
BACKGROUND: [atmospheric depth]
ATMOSPHERE: [lighting with source, color, shadows]
EMOTIONAL TONE: [single evocative phrase]
NEGATIVE: clay, claymorphic, pastel, chrome robot, sterile geometry, generic brain, flat clipart, cartoon, stock photo, [add specifics]

Critical rule: Every prompt MUST have an organic anchor. If you can't name the organic element, the prompt will fail council review.

STEP 3: COUNCIL

Invoke the visual-creation-council agent as a Task:

Use the Task tool with subagent_type="general-purpose":
"You are the Visual Creation Council for FrankX. Read .claude/agents/visual-creation-council.md for your full scoring criteria. Then review this prompt: [paste filled template]. Produce the full council review with scores per lens and a binding verdict."

BLOCKING: Do not proceed without APPROVED verdict.

  • APPROVED (>=7.5, no vetoes) → proceed to Step 4
  • NEEDS-REVISION (6.0-7.4) → apply ALL feedback, re-draft, re-submit (max 3 cycles)
  • REJECTED (<6.0 or any veto) → fundamentally rework the concept

STEP 4: TEST (Human Gate)

Generate exactly 1 image using the approved prompt via MCP:

mcp__nanobanana__generate_image({
  prompt: "[composed prompt from template]",
  aspect_ratio: "16:9",        // or 1:1, 9:16 as needed
  model_tier: "pro",
  enable_grounding: false,
  thinking_level: "high",
  resolution: "high",
  negative_prompt: "[from template]",
  output_path: "[target path in public/images/]"
})

After generation:

  1. Read the generated image using the Read tool to show it inline
  2. Score honestly against 7 gates + organic warmth test:
    • Gate 1: Brand Alignment (matches style?)
    • Gate 2: Color Balance (3+ accents visible?)
    • Gate 3: Information Density (every area purposeful?)
    • Gate 4: Depth Layers (foreground/mid/background?)
    • Gate 5: Text Legibility (labels readable at 680px?)
    • Gate 6: Icon Quality (rich 3D, not flat?)
    • Gate 7: Scroll-Stop (would you pause scrolling?)
    • Organic Warmth: living presence or sterile chrome?
  3. Log to data/image-generation-log.json (append entry with full metadata)
  4. Ask the user: "Continue as-is, iterate on [aspect], or discard and restart?"

BLOCKING: Do NOT proceed until user responds.

STEP 5: ITERATE (If Needed)

If user requests changes:

  1. Apply feedback to the prompt template (modify specific fields)
  2. If changes are significant (new metaphor, style switch): re-submit to council (Step 3)
  3. If changes are minor (color tweak, position shift): generate another test
  4. Max 2 iteration cycles before proceeding

STEP 6: EXECUTE (Batch)

If user approves and wants variations or a batch:

  1. Generate 3-5 images max with minor prompt variations
  2. Score each against 7 gates + organic warmth
  3. Log all to image-generation-log.json
  4. Show each image inline with honest score
  5. Flag any scoring < 6/8 gates for regeneration

Key Rules

  1. NEVER batch 24 images at once — max 5 per batch, always with test first
  2. NEVER cheerleader — if an image scores 5/8 gates, say 5/8, not "looking great"
  3. ALWAYS show images inline — use Read tool after every generation
  4. ALWAYS log — no generation goes untracked
  5. Organic-first or fail — every prompt needs a living anchor before any tech layer
  6. Human gate at Step 4 — no batch without explicit user approval of test image

Part of the Visual Creation Council system. Quality over quantity.
