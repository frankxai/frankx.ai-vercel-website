/design-gods — Design Excellence Orchestrator v2.0

Purpose: Invoke the full design excellence swarm for FrankX. Routes requests to the right specialist agent(s) based on task type. Ensures every design decision meets brand, accessibility, and technical standards.

Power: 5 specialist agents + 7-layer token system + 5-gate QA + 7-gate visual filter. The equivalent of a 100-person senior design team distilled into an orchestrated swarm.

What This Command Does

  1. Classifies the design request into one or more specialist domains
  2. Invokes the relevant agents (brand-architect, ui-engineer, design-qa, visual-director, figma-bridge)
  3. Sequences work correctly (design → build → QA, never QA → design)
  4. Synthesizes outputs into a single actionable response
  5. Validates against the full design system before shipping

Invocation Modes

/design-gods audit [page or component]

Full brand + quality audit. Invokes brand-architect → design-qa. Output: Brand Audit Report + Design QA Report with file:line fixes.

/design-gods build [component description]

New component from design brief. Invokes ui-engineer → design-qa. Output: Complete TypeScript component + QA verdict. Zero hardcoded values guaranteed.

/design-gods figma [URL]

Implement a Figma design. Invokes figma-bridge → ui-engineer → design-qa. Output: Component code mapped to FrankX tokens + QA verdict.

/design-gods visual [brief]

Generate visual asset. Invokes visual-director (7-gate) → nanobanana. Output: Direction brief + generated image(s) + asset registry entry.

/design-gods system

Full design system health check. Comprehensive scan of ALL components.

SYSTEM HEALTH CHECK PROTOCOL:
1. Scan all files in components/ and app/ for token violations
2. Count hardcoded hex, wrong glass opacity, wrong radius, wrong font
3. Check motion: transition-all, missing reduced-motion, wrong duration
4. Check accessibility: missing focus rings, small touch targets, missing aria
5. Check brand: wrong character placement, banned voice terms, weak CTAs
6. Generate health score (0-100) with breakdown by gate
7. Rank top 10 files needing fixes (by violation count)

Output: System Health Dashboard + Top 10 priority fixes.

/design-gods token-audit

Deep token compliance scan. Finds every hardcoded value in the codebase.

TOKEN AUDIT PROTOCOL:
1. Grep all .tsx files for hardcoded hex patterns (#[0-9a-fA-F]{3,8})
2. Grep for hardcoded rgba() patterns
3. Grep for bg-white/ with opacity > 0.08
4. Grep for text-gray-* (should be text-white/ with opacity)
5. Grep for shadow-lg, shadow-md (should be elevation system)
6. Grep for rounded-lg on card-like components (should be rounded-4xl)
7. Grep for transition-all (should be specific property)
8. Grep for font-poppins on non-heading elements
9. Output: violation list with exact file:line + suggested fix

/design-gods migrate [file or pattern]

Migrate existing components to full token compliance.

MIGRATION PROTOCOL:
1. Read target file(s)
2. Identify all token violations (run mini token-audit)
3. For each violation: apply the correct token replacement
4. Verify: re-read file, confirm zero violations
5. Run 5-gate QA on migrated file
6. Output: diff summary + QA verdict

/design-gods a11y [page or component]

Focused accessibility audit (WCAG 2.2 + Apple HIG).

ACCESSIBILITY AUDIT:
1. Interactive elements: focus-visible ring present?
2. Touch targets: all ≥44×44px?
3. Text contrast: ≥4.5:1 (AA) on all text?
4. Animations: prefers-reduced-motion check?
5. Images: alt text on non-decorative?
6. Forms: labels associated with inputs?
7. Keyboard: all interactive elements reachable?
8. Screen readers: ARIA landmarks, live regions?

Agent Invocation Protocol

Task: design-gods [mode] [args]

STEP 1 — Classify (orchestrator decision, no agent needed):
  Read the request → map to classification table → decide routing

STEP 2 — Load Context:
  Read lib/design/tokens.ts (surface, elevation, stateLayer, focusRing, etc.)
  Read lib/design/motion.ts (m3Curves, motionDurations, motionSpring, variants)
  Read the target files

STEP 3 — Invoke Primary Agent (blocking):
  Use Task tool with subagent_type matching agent name
  Wait for output

STEP 4 — Invoke Secondary Agent (if applicable):
  Pass primary output as context
  Wait for QA verdict or synthesis

STEP 5 — Synthesize (orchestrator):
  Combine outputs into single response
  List files created/modified
  List next actions

STEP 6 — Design QA (always last):
  If any UI code was written: run design-qa 5-gate validation
  BLOCKED verdict = do not ship, fix immediately
  NEEDS REVISION = fix before merge
  CONDITIONAL = ship with fix ticket
  PASS = ship it

Design Standards Enforced

Non-Negotiable Gates

  • ✅ All colors via tokens (never hardcoded hex)
  • ✅ Glass opacity 0.02–0.08 maximum
  • ✅ Focus rings on all interactive elements (focusRing.default)
  • ✅ Touch targets ≥44px (Apple HIG, touchTarget.standard)
  • ✅ M3 state layers on cards/buttons (stateLayer.hover/pressed/focused)
  • ✅ M3 motion curves (m3Curves.emphasized, NOT css ease-in-out)
  • ✅ prefers-reduced-motion check on all animations
  • ✅ Brand character matches page domain (per character-to-domain map)
  • ✅ Fluid typography (text-fluid-* preferred over fixed breakpoints)
  • ✅ Elevation system (elevation.sm/md/lg, NOT shadow-lg)
  • ✅ Semantic gradients (gradients.text.brand, NOT custom bg-gradient)

7-Layer Token System

Layer 1: Color Primitives    → emerald-500, purple-600, etc.
Layer 2: Semantic Colors     → --brand-primary, --text-primary, etc.
Layer 3: Surfaces            → surface.glass, surface.elevated, etc.
Layer 4: Elevation           → elevation.sm, elevation.md, elevation.brand
Layer 5: Typography          → font-display, font-sans, text-fluid-*
Layer 6: Spacing             → 4px grid, sectionSpacing.*, componentPadding.*
Layer 7: Motion              → m3Curves.*, motionDurations.*, motionSpring.*

Usage rules

  1. Always use _thumb.jpeg for avatars/thumbnails — 10x smaller, same quality at small sizes
  2. Use .png only for large hero images where detail matters
  3. Apply object-position: object-top for chibi variants (head-heavy compositions)
  4. Add drop-shadow-[0_0_60px_rgba(59,130,246,0.3)] for floating character appearance
  5. Ghost/watermark use: opacity-10 or opacity-15 on background decorative placements
