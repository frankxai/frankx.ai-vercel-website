/music-import - Music Catalog Management

Manage the FrankX music inventory — add, scrape, clean, and sync tracks.

Quick Commands

When the user provides a Suno URL or track ID:

  1. Extract the sunoId from the URL (format: https://suno.com/song/{sunoId})
  2. Fetch track metadata via the Suno CDN/API
  3. Add to data/inventories/frankx/music.json using the schema below
  4. Confirm the addition

Operations

1. Add Single Track

Trigger: User provides a Suno URL like https://suno.com/song/abc123-def456

# Extract sunoId from URL
SUNO_ID="abc123-def456"

# Verify the track exists (CDN check)
curl -sI "https://cdn1.suno.ai/${SUNO_ID}.mp3" | head -1

Then add to inventory with this schema:

{
  "id": "track-slug-name",
  "type": "music",
  "title": "Track Title",
  "brand": "frankx",
  "status": "published",
  "tags": ["genre1", "genre2"],
  "platform": "suno",
  "sunoId": "the-uuid",
  "sunoUrl": "https://suno.com/song/the-uuid",
  "genre": ["genre1", "genre2"],
  "plays": 0,
  "section": "songs"
}

Optional fields: coverUrl, likes, duration, audioUrl, mood, description, promptUsed

2. Bulk Scrape Profile

Trigger: User says "scrape", "sync", "discover new tracks", or "update catalog"

cd /mnt/c/Users/Frank/FrankX/.worktrees/vercel-ui-ux
node scripts/scrape-suno-catalog.mjs --index-only

After scraping, report:

  • How many new tracks discovered
  • How many existing tracks updated
  • Total inventory count

3. Clean Genre Data

Trigger: User says "clean genres", "fix tags", or "clean up catalog"

Read data/inventories/frankx/music.json and for each track:

  • If genre array contains long strings (>30 chars), they're raw Suno prompt descriptions
  • Extract only genre keywords: e.g., "K-pop", "Electronic", "Ambient", "Lo-fi", "Orchestral"
  • If tags duplicates genre, deduplicate
  • If title starts with "Track " followed by a hash, flag it for manual naming

4. Download + Upload MP3s

Trigger: User says "download mp3s", "upload to blob", or "self-host tracks"

cd /mnt/c/Users/Frank/FrankX/.worktrees/vercel-ui-ux

# Download only
node scripts/scrape-suno-catalog.mjs --download

# Download + upload to Vercel Blob
node scripts/scrape-suno-catalog.mjs --download --upload

Requires BLOB_READ_WRITE_TOKEN env var for upload.

5. Stats Report

Trigger: User says "music stats", "catalog status", or "how many tracks"

Read data/inventories/frankx/music.json and report:

  • Total tracks
  • Tracks by section (songs, hooks, featured)
  • Tracks with/without audioUrl (self-hosted vs CDN)
  • Tracks with/without coverUrl
  • Tracks with empty genre
  • Profile stats (_profileStats)
  • Last updated date

Genre Keyword Reference

Clean genre tags should use these standard keywords: ambient, cinematic, electronic, epic, folk, hip-hop, indie, jazz, k-pop, lo-fi, meditation, metal, orchestral, pop, r&b, rock, soul, synthwave, trap, world

After Adding Tracks

  1. Commit the updated music.json
  2. Push to production: git push origin main
  3. Vercel auto-deploys from main
