/mcp-optimize — MCP Server Load Optimization

Purpose: Audit MCP server load per session, identify redundant/unused servers, and recommend configuration changes to cut per-session memory from ~1.5GB to ~400MB.

Input

  • Mode: $ARGUMENTS (default: audit | audit | profiles | apply-light | apply-full)
    • audit — show current MCP server load and per-plugin memory usage
    • profiles — show recommended profiles for different task types
    • apply-light — suggest disabling unused plugins (non-destructive)
    • apply-full — restructure MCP config for optimal per-task loading
    • (no args) — full audit + profiles + recommendations

MCP Architecture Knowledge

Three Layers of MCP Servers

Every Claude Code session loads servers from three layers:

Layer 1: Project-scoped (~/.claude.json)     ← LOCAL processes
  memory, resend, sequential-thinking, v0,
  nanobanana, n8n-mcp, mcp-doctor
  = 7 servers × ~40MB each = ~280MB

Layer 2: Plugins (installed_plugins.json)    ← LOCAL processes
  claude-mem (chroma-mcp: 200-300MB!)
  6× agentic-creator-skills sub-plugins
    each spawns: Notion KB + Slack + Resend
  = ~8-15 servers × ~40MB + chroma ~250MB = ~600-900MB

Layer 3: Claude.ai built-in MCP              ← SERVER-SIDE (free!)
  Figma, Vercel, Notion, Slack, Zapier, Miro
  = 0MB local (runs on Anthropic infrastructure)

Total per session: ~880-1,180MB of local processes. With 5 sessions: ~4.4-5.9GB just for MCP.

The Duplication Problem

The agentic-creator-skills plugins are the worst offenders:

Plugin Notion KB Slack Resend Unique Value
core Yes Yes No Base routing
content-engine Yes Yes Yes Content creation
brand-architect Yes No No Brand voice
intelligence No (graph) Yes No Graph memory
music-lab Yes No No Music context
product-launcher Yes Yes Yes Product launch
visual-studio No No No Image gen

5 plugins load Notion. 4 load Slack. 2 load Resend. That's 11 redundant connections — each spawning a separate process.

Meanwhile, Claude.ai already provides Notion + Slack built-in (Layer 3) at zero local cost.

Audit Phase

Run these diagnostics:

# Count running MCP-related processes
echo "MCP process count:"
ps aux | grep -cE "(chroma-mcp|mcp-server|uvx|npx.*mcp)"

# Memory usage per MCP type
echo "chroma-mcp instances and memory:"
ps aux | grep "[c]hroma-mcp" | awk '{sum+=$6; count++} END {print count, "instances,", int(sum/1024), "MB total"}'

echo "Node MCP servers:"
ps aux | grep "[n]ode" | grep -i mcp | awk '{sum+=$6; count++} END {print count, "processes,", int(sum/1024), "MB total"}'

Optimization Actions

Quick Win: Disable Unused Plugins

Run in Claude Code:

/plugins

Then disable these if not actively using them:

  • music-lab@agentic-creator-skills (unless making music)
  • visual-studio@agentic-creator-skills (nanobanana handles images)
  • product-launcher@agentic-creator-skills (unless launching a product)
  • brand-architect@agentic-creator-skills (brand rules in SKILL.md instead)

Each disabled plugin saves ~80-160MB per session.

Medium Win: Restructure Global Config

Move situational servers from global (~/.claude.json) to project .mcp.json with "disabled": true:

{
  "mcpServers": {
    "n8n-mcp": {
      "command": "/home/frankx/.nvm/versions/node/v24.12.0/bin/n8n-mcp",
      "disabled": true,
      "_note": "Enable with: claude mcp enable n8n-mcp"
    },
    "resend": {
      "command": "npx",
      "args": ["-y", "@resend/mcp"],
      "disabled": true,
      "_note": "Enable for newsletter/email work"
    }
  }
}

Big Win: Fix chroma-mcp Duplication

The claude-mem worker-service.cjs spawns a new chroma-mcp per session connection via uv tool uvx --python 3.13 chroma-mcp --client-type persistent.

Option A: Set environment variable for HTTP mode (shared server):

export CLAUDE_MEM_CHROMA_HOST=localhost
export CLAUDE_MEM_CHROMA_PORT=8000
# Then run chroma server separately: chroma run --port 8000

Option B: Report bug to thedotmack/claude-mem repo.

Option C: Periodically clean zombie chroma-mcp processes:

# Kill chroma-mcp processes older than 2 hours with 0% CPU
ps -eo pid,etimes,pcpu,comm | awk '$4=="python" && $2>7200 && $3==0.0 {print $1}' | xargs -r kill 2>/dev/null

Safety Rules

  1. Never remove a server without checking if current session uses it
  2. disabled: true is always preferred over removing config entries
  3. Layer 3 (claude.ai built-in) is free — use Notion/Slack/Figma from there instead of plugins
  4. Always run /mcp after config changes to verify server status
  5. Plugin disabling requires /plugins command — can't be done via file edits
  6. Test one change at a time — disable one plugin, verify session still works
