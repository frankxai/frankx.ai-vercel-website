Video Produce

Purpose: Generate a full video production package for a given topic, ready for recording and publishing.

Input

The user provides:

  1. Topic (required) — the subject of the video (e.g., "Building multi-agent systems with ACOS")
  2. Format (optional) — tutorial, breakdown, opinion, showcase, deep-dive. Defaults to deep-dive.
  3. Target length (optional) — in minutes. Defaults to 12.

Process

Step 1: Load Context

Read the following files before generating anything:

  • docs/strategy/CONTENT_VOICE_DNA.md — voice rules (CRITICAL)
  • docs/strategy/IMAGE_STYLE_DNA.md — thumbnail visual style
  • data/video-watchlists.json — category alignment for tags
  • data/video-vault-100.json — existing vault for topic overlap check

Confirm the topic does not duplicate an existing vault entry. If overlap detected, note it and suggest a differentiated angle.

Step 2: Determine Category

Map the topic to one of the watchlist categories:

  • AI Engineering, AI Agents, AI Tools, AI Fundamentals
  • Creative AI, Music Production
  • Strategy, Creator Economy, Learning
  • Mindset, Rituals, Creativity

Step 3: Generate Script Structure

Produce a structured script with these sections:

=== VIDEO SCRIPT: {Title} ===
Format: {format} | Target: {minutes} min | Category: {category}

--- HOOK (0:00 - 0:30) ---
[Opening statement — bold claim, surprising stat, or direct question.
Must grab attention in the first 5 seconds. No preamble. No "Hey guys."
Lead with the most interesting result or insight.]

--- INTRO (0:30 - 1:30) ---
[Context — what this video covers and why it matters.
State the problem clearly. Mention what the viewer will walk away with.
Keep it under 60 seconds. No filler.]

--- MAIN CONTENT ---

Section 1: {Title} (1:30 - {timestamp})
[Core concept or demonstration.
Include specific data points, code snippets, or screen references.
Each section should deliver one clear takeaway.]

Section 2: {Title} ({timestamp} - {timestamp})
[Build on Section 1. Go deeper or show the next step.
Include a "here's what most people get wrong" moment if applicable.]

Section 3: {Title} ({timestamp} - {timestamp})
[Advanced application, real-world example, or architecture walkthrough.
Show results — metrics, output, before/after.]

--- RECAP ({timestamp} - {timestamp}) ---
[3-5 bullet summary of key takeaways.
No new information. Reinforce the hook's promise.]

--- CTA ({timestamp} - end) ---
[One clear action. Link to blog post, ACOS repo, or product page.
No begging for likes. One line: "Link below. Fork it." or similar.]

Adjust section count based on target length:

  • Under 8 min: 2 main sections
  • 8-15 min: 3 main sections
  • Over 15 min: 4-5 main sections

Full prompt for image generation:

Thumbnail rules:

  • NEVER generic brain imagery
  • NEVER single-color saturation
  • NEVER flat clipart or sparse composition
  • ALWAYS 3+ accent colors, 3+ depth layers, glass/chrome materials
  • Text on thumbnail: 2-4 words maximum, must be readable at mobile size

Step 5: Generate YouTube Tags (30)

=== YOUTUBE TAGS (30) ===

Tag rules:

  • First tag matches the video title exactly (lowercase)
  • Include 3-5 long-tail phrases (3+ words)
  • Mix specific tools (e.g., "claude code tutorial") with broad terms (e.g., "ai engineering")
  • Always include: frankx, frankx ai, acos
  • No trademarked terms used misleadingly

Step 8: Terminal Output

Display the complete production package in the terminal, formatted for readability. Include:

  • Character count for description (must be under 5,000)
  • Tag count confirmation (target: 30)
  • Estimated recording time based on word count (~150 words/min)

Voice Rules (CRITICAL)

READ FIRST: docs/strategy/CONTENT_VOICE_DNA.md

The Voice: Systems architect explaining his workshop to a peer. Precise, data-backed, dry humor, respects the viewer's intelligence.

DO:

  • Lead with results and specific numbers
  • Use the fragment-declaration rhythm
  • Reference real tools, real metrics, real architecture
  • "Here's what I built. Here's how. Fork it."

DON'T:

  • "Hey guys, welcome back to another video"
  • "Smash that like button"
  • "In today's video we're going to..."
  • Spiritual/guru language
  • Emojis anywhere in the script
  • "Game-changer", "revolutionary", "unlock"
  • Over-explain what they're about to see — just show it
