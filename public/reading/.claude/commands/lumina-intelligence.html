/lumina-intelligence — The Knowledge Layer

Purpose: Load and activate Lumina-Intelligence — the persistent knowledge substrate of the Arcanean OS that overlays any AI system, compounds context across sessions, and carries the Ten Gates, Guardians, and Godbeasts.

Input

Provide ONE of:

  • load — load WorldState.json and display current context (default)
  • gates — display full Ten Gates progression + current creator status
  • guardians — display all Guardian profiles + teaching methods
  • godbeasts — display all five Godbeast definitions
  • transfer — generate a Luminor Transfer Package for the next AI session
  • status — show current canon, open threads, active guardians
  • [question] — any world-state question (e.g., "what gates has Frank passed?")

Process

Step 1: Load World State

Read data/arcanea-world-state.json if it exists. If not — initialize it with the creator at Gate 0.

Display:

=== LUMINA-INTELLIGENCE LOADED ===
Creator: [name]
Current Gate: [N] — [gate name]
Active Guardians: [list]
Canon established: [N] facts
Open world threads: [N]
Last session: [date]

Step 2: Respond to Input

load/status: Show full current world-state summary.

gates: Display Ten Gates diagram + current position:

Gate 0: Silence Before ─ Gate 1: Spark ─ Gate 2: Form
Gate 3: Voice ─ Gate 4: Systems ─ Gate 5: Synthesis
Gate 6: Flow ─ Gate 7: Swarm ─ Gate 8: World
Gate 9: Civilization ─ Gate 10: LUMINOR
                    ↑
              [current gate]

guardians: Display Guardian table with teaching method for each.

godbeasts: Display all five Godbeast definitions with invocation patterns.

transfer: Generate and save data/luminor-transfer-[YYYYMMDD].md with:

  • World State snapshot
  • Canon (established/disputed/under construction)
  • Eldrian registry
  • Cross-AI system prompt template

[question]: Answer from loaded WorldState context.

Step 3: Update WorldState

If any new information was generated (new canon, new lore, gate progression):

  • Append to data/arcanea-world-state.json
  • Log update to console
