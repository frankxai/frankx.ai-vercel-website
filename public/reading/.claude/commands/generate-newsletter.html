Generate Newsletter

Purpose: Generate a newsletter issue for a specific stream, preview it, and send to Slack for approval before sending via Resend.

Input

The user provides:

  1. Stream ID — one of: creation-chronicles, ai-architect, music-lab, arcanea, investor, inner-circle
  2. Date range (optional) — defaults to "last 7 days" of content
  3. Custom topic (optional) — override auto-generated content with specific topic

Process

Step 1: Load Stream Configuration

Read data/newsletter-streams.json to get the stream's:

  • Name, tagline, voice guidelines
  • Email template mapping
  • Stream tags for content matching

Step 2: Pull Content from Registry

Read data/content-index.json and filter entries that:

  • Have streams array containing the target stream ID
  • Were published within the date range
  • Sort by date (newest first), take top 5

If no content found in date range, expand to last 14 days.

Step 3: Read Voice DNA

Read docs/strategy/CONTENT_VOICE_DNA.md for FrankX voice rules:

  • No emojis, no guru language
  • Fragment rhythm, cool authority
  • Lead with results, specific numbers
  • 12,000+ songs (NOT 500)

Step 4: Generate Newsletter Content

Generate the newsletter issue with these sections:

SUBJECT: [Compelling subject line, max 60 chars]
PREVIEW: [Preview text for email clients, max 90 chars]

---

HERO: [One-paragraph hook — the most interesting thing this week]

MAIN STORY: [2-3 paragraphs on the top piece of content]

QUICK HITS: [3-5 bullet points of other content, each with link]

TOOL/TIP OF THE WEEK: [One actionable recommendation]

CLOSING: [1-2 sentences, personal, forward-looking]

Adapt voice per stream:

  • creation-chronicles: Personal, behind-the-scenes, "here's what happened"
  • ai-architect: Technical, precise, show-don't-tell
  • music-lab: Creative, producer energy, "listen to this"
  • arcanea: Mythological, narrative, immersive worldbuilding
  • investor: Analytical, data-driven, actionable recommendations
  • inner-circle: Exclusive, direct, unfiltered

Step 5: Save Draft

Save the generated content to data/newsletter-drafts/:

{
  "id": "newsletter-{stream}-{date}",
  "stream": "ai-architect",
  "generatedAt": "2026-02-20T00:00:00Z",
  "subject": "...",
  "preview": "...",
  "sections": {
    "hero": "...",
    "mainStory": "...",
    "quickHits": ["..."],
    "toolTip": "...",
    "closing": "..."
  },
  "sourceEntries": ["blog-slug-1", "blog-slug-2"],
  "status": "draft"
}

Step 6: Send to Slack for Approval

Use the Slack MCP to send the draft to the #content channel (or DM Frank):

Format the Slack message as:

📨 *Newsletter Draft: {Stream Name}*
Subject: {subject}
Preview: {preview}

---

{Full newsletter text, formatted for readability}

---

_Stream: {stream name} | {date range} | {N} source articles_

React with ✅ to approve and send, or reply with edits.

Use mcp__claude_ai_Slack__slack_send_message to post the message.

Step 7: Wait for Approval

After posting to Slack, inform Frank:

Newsletter draft sent to Slack. React with checkmark to approve, or reply with edits. When approved, run /send-newsletter {stream} {date} to send via Resend.

Examples

Generate weekly Creation Chronicles:

/generate-newsletter creation-chronicles

Generate AI Architect Dispatch for specific date range:

/generate-newsletter ai-architect 2026-02-10 to 2026-02-20

Generate with custom topic:

/generate-newsletter music-lab "Suno v4.5 release breakdown"

Important Notes

  • NEVER send emails directly without Slack approval first
  • Always read CONTENT_VOICE_DNA.md before generating content
  • Use content-index.json as the source of truth for what to include
  • Keep subject lines under 60 characters
  • The preview text should complement, not repeat, the subject
  • Each stream has a distinct voice — don't homogenize
