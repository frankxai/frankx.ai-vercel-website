1. Brand Attributes

FrankX has five core brand attributes. Every piece of design, copy, and interaction should express a combination of these qualities. Different contexts emphasize different attributes, but all five should be present in the overall brand experience.

The Five Attributes

Precise

Technical depth. Attention to detail. Every pixel, every word, every interaction is intentional.

Layer Expression
Copy Exact numbers, specific workflows, verifiable claims. "This pattern handles 90% of multi-agent needs" not "comprehensive solution."
Design Pixel-perfect alignment. Consistent spacing. No orphaned text. Mathematical proportions.
Code Type-safe. Zero warnings. Clean architecture. Named exports.
Experience Fast load times. Smooth transitions. No layout shift.

Bold

Confident without being loud. Authority through craft, not claims. Striking when it matters.

Layer Expression
Copy "Build AI Systems That Actually Ship." Strong verbs. No hedging. Lead with the result.
Design High contrast. Large type for headlines. Generous negative space that says "we don't need to fill this."
Code Opinionated architecture choices. Clear conventions. Decisive.
Experience Hero sections that command attention. Unmistakable brand presence in 3 seconds.

Genuine

Real work, real results, no fluff. Frank builds in public. The brand reflects honest exploration.

Layer Expression
Copy "Here's the exact workflow I use" not "unlock your potential." First person. Specific. Studio energy.
Design Clean layouts. No decorative noise. Photography over stock. Real code over mockups.
Code Show the actual implementation. Comments explain "why" not "what."
Experience Real metrics. Actual project screenshots. Working demos over slideshows.

Creative

AI-augmented. Multi-modal. Pushing boundaries across music, code, writing, and design.

Layer Expression
Copy Bridge technical and creative. "500+ AI songs" sits next to "Enterprise AI Architecture."
Design Aurora gradients. Organic shapes meeting geometric precision. Unexpected color combinations that work.
Code Elegant solutions. Creative problem-solving. Beautiful APIs.
Experience Surprise moments. Easter eggs. The feeling of discovering something crafted with care.

Warm

Accessible despite depth. A living legacy for family. Inviting people in, not gatekeeping.

Layer Expression
Copy "explaining to a friend at 2am." Complex ideas made accessible without dumbing down.
Design Rounded corners. Warm accent colors (gold, amber). Readable body text. Light humor in microcopy.
Code Well-documented. Generous README files. Thoughtful error messages.
Experience Clear navigation. Helpful empty states. Progressive disclosure. Never overwhelming.

Application by Context

Context Primary Secondary Tertiary
Blog / Technical Content Precise Genuine Bold
Product Pages Bold Creative Warm
Music / Creative Work Creative Bold Genuine
Documentation / Guides Precise Warm Genuine
Social Media Bold Creative Warm
Developer Tools / ACOS Precise Bold Creative
Personal / Family Pages Warm Genuine Creative

2. Logo System

Primary Mark: The FX Monogram

The FrankX mark is a geometric monogram combining the letters F and X. It represents the intersection of Frank's identity — where technical precision (the structured F) meets creative exploration (the dynamic X).

Construction

  • Built on a square grid (24x24 base units)
  • The F is structured, vertical, grounded
  • The X cuts through with forward momentum
  • Corner radius: 2 units (8.3% of height)
  • Stroke weight: 2.5 units

Clear Space

  • Minimum clear space: 25% of mark height on all sides
  • For circular crops, center the mark and balance top-to-bottom visually

Wordmark Lockup

The full FrankX logo pairs the FX monogram with the wordmark.

[FX] FrankX          ← Horizontal lockup (default)

[FX]
FrankX               ← Stacked lockup (narrow contexts)

Sizing

  • Minimum size (digital): 24px height for monogram, 80px for full lockup
  • Minimum size (print): 12mm height for monogram
  • Clear space of 50% mark height for full lockup

Color Variants

On Dark Backgrounds (Primary)

  • Mark: White (#FFFFFF)
  • Wordmark: White (#FFFFFF)
  • This is the default. FrankX is a dark-mode-first brand.

On Light Backgrounds

  • Mark: Deep Navy (#0F172A)
  • Wordmark: Deep Navy (#0F172A)

On Mid-tone Backgrounds

  • Mark: White (#FFFFFF) with subtle drop shadow (0 2px 8px rgba(0,0,0,0.3))
  • Ensure minimum 4.5:1 contrast ratio

Accent Usage

  • The mark may appear in Cosmic Purple (#AB47C7) or Aurora Blue (#43BFE3) for special contexts
  • Only on backgrounds where contrast ratio meets WCAG AA (4.5:1)

Don'ts

Rule Why
Don't crop the mark Loses recognition
Don't rotate The F needs vertical grounding
Don't stretch or distort Breaks geometric precision
Don't place on busy backgrounds without contrast Readability first
Don't use brand colors on the wordmark while mark is different Keep lockup unified
Don't add effects (glow, shadow, gradient fill) The mark is clean geometry
Don't scale the mark to be the primary visual Let content lead
Don't recreate in a different font Use the official assets only

3. Brand Tagline

"Build what matters."

This tagline captures FrankX in four words:

  • Build — action-oriented, maker energy
  • What matters — intentional, quality over quantity, legacy over vanity

Usage Rules

  • Use as a headline or anchor, not filler text
  • Placement: below the logo in hero sections, or as a standalone statement
  • Never modify ("Build what matters to you" — no. The brevity is the point.)
  • Always in sentence case: "Build what matters." — never ALL CAPS

Tagline Lockup

[FX] FrankX
Build what matters.

Clear space: 50% of mark height between wordmark and tagline.

Don'ts

  • Don't overlay on complex imagery where readability suffers
  • Don't pair with the logo at very small sizes (below 120px lockup width)
  • Don't use as body copy or within paragraphs

6. Typography

Font Stack

Primary: Inter

The workhorse. Used for body copy, UI elements, navigation, metadata.

font-family: 'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
  • Why Inter: Optimized for screens. Excellent at small sizes. Variable font with fine-tuned optical sizing. Built for technical content.
  • Weights used: 400 (Regular), 500 (Medium), 600 (SemiBold)

Display: Poppins

Headlines and display text. Adds warmth and personality to Inter's precision.

font-family: 'Poppins', 'Inter', sans-serif;
  • Why Poppins: Geometric, modern, slightly rounded — adds the "warm" attribute. High readability at large sizes.
  • Weights used: 600 (SemiBold), 700 (Bold)

Serif: Playfair Display

Reserved for editorial and premium contexts only.

font-family: 'Playfair Display', Georgia, serif;
  • Where: Blockquotes in blog posts (italic), book chapter headings, premium marketing
  • Weight: 400 Italic only
  • Never use for UI, navigation, or buttons

Monospace: JetBrains Mono

Code blocks, terminal output, technical specifications.

font-family: 'JetBrains Mono', 'Fira Code', 'Cascadia Code', monospace;
  • Features: Ligatures enabled for code (!=, =>, ===)
  • Weights used: 400 (Regular), 500 (Medium)

7. Illustration & Visual Language

Visual System: Aurora Architecture

FrankX's visual language is built on three pillars:

1. Glassmorphism

Frosted glass surfaces that create depth and hierarchy.

/* Standard glass card */
background: rgba(255, 255, 255, 0.03);
backdrop-filter: blur(12px);
border: 1px solid rgba(255, 255, 255, 0.06);
border-radius: 16px;

/* Elevated glass */
background: rgba(255, 255, 255, 0.05);
backdrop-filter: blur(20px);
border: 1px solid rgba(255, 255, 255, 0.1);
box-shadow: 0 8px 32px rgba(0, 0, 0, 0.3);

Rules:

  • Always visible against the dark background (min opacity 0.03)
  • In light mode, use rgba(0,0,0,0.03) and bg-white/80+
  • Maximum blur: 20px (more becomes undefined)
  • Never stack more than 2 glass layers

2. Aurora Gradients

Organic, flowing color fields that suggest intelligence and creativity.

Patterns:

Nebula    → Purple to Blue (cosmic, AI intelligence)
Horizon   → Blue to Emerald (technology meeting growth)
Solar     → Gold to Purple (excellence meeting depth)
Full Arc  → Purple → Blue → Emerald (complete brand expression)

Application:

  • Section dividers (thin gradient line, 2px height)
  • Card hover states (subtle gradient border)
  • Hero backgrounds (as radial gradient, low opacity: 0.05-0.15)
  • CTA buttons (linear gradient fill for primary actions)

Don'ts:

  • Never use gradients on body text
  • Never layer gradient over gradient
  • Gradient backgrounds must be subtle (max 15% opacity over dark)
  • Never use rainbow-order gradients (cheap-looking)

3. Constellation Patterns

Connected node networks representing intelligence, systems, and interconnection.

Elements:

  • Nodes: 4-8px circles, Slate 600 or accent color
  • Connections: 1px lines, Slate 700 opacity
  • Active nodes: Accent color with subtle glow
  • Clusters: Groups of 5-15 nodes

Application:

  • AI World page background
  • Hero section subtle backgrounds
  • Research hub visualization
  • Diagram backgrounds

Rules:

  • Constellations are always background elements (never foreground)
  • Max opacity: 0.3
  • Animated constellations: subtle drift only, no rapid movement
  • Responsive: simplify or hide on mobile

Photography Direction

Subjects:

  • Developer workspaces (monitors, code, minimal)
  • Amsterdam cityscape (occasional, authentic)
  • Abstract technology (server rooms, fiber optics)
  • Nature + tech intersections (organic meeting digital)

Treatment:

  • Slightly desaturated, leaning cool
  • High contrast
  • Dark vignette edges blend into brand backgrounds
  • Never stock-looking (no pointing at screens, no handshakes)

When to Use Photography:

  • About page
  • Blog headers (when relevant)
  • Social media cards
  • Never on product pages (use illustration/UI instead)

Infographic Style

Stats & Metrics

Metric name here
Supporting context
  • Icon: Lucide, 24px, accent color
  • Number: Poppins 600, Display M, White
  • Label: Inter 500, Body S, Slate 400

8. Icons

System: Lucide React

FrankX uses Lucide React as its exclusive icon system.

Why Lucide

  • Open source, MIT licensed
  • Consistent 24px grid, 2px stroke
  • 1,500+ icons covering all needs
  • React-native components with tree-shaking
  • Maintained by the community (forked from Feather Icons)

Sizes

Size Pixels Usage
XS 16px Inline with text, badges
SM 20px Buttons, input adornments
MD 24px Default, navigation, cards
LG 32px Feature highlights
XL 48px Hero sections, empty states

Color Rules

  • Navigation icons: Slate 400 (default), White (active/hover)
  • Content icons: Match surrounding text color
  • Accent icons: Use domain color (see Color System)
  • Interactive icons: Slate 400 → White on hover (150ms transition)

Stroke Width

  • Default: 2px (Lucide standard)
  • Small contexts (16px): 1.5px for legibility
  • Never use filled icons — FrankX is a stroke-icon brand

Icon Don'ts

Rule Why
Never use emoji as icons Inconsistent rendering, unprofessional
Never mix icon libraries Visual inconsistency
Never use icons without a text label in navigation Accessibility
Never scale icons beyond 48px Use illustration instead
Never add custom stroke colors to standard icons Brand consistency

9. Layout & Grid

Grid System

Desktop (1280px+)

  • Container: max-w-7xl (1280px) centered
  • Columns: 12-column grid
  • Gutter: 24px (gap-6)
  • Page margin: 32px (px-8)

Tablet (768px - 1279px)

  • Container: Full width with padding
  • Columns: 8-column grid or 2-column layout
  • Gutter: 20px (gap-5)
  • Page margin: 24px (px-6)

Mobile (< 768px)

  • Container: Full width with padding
  • Columns: Single column
  • Gutter: 16px (gap-4)
  • Page margin: 16px (px-4)

Spacing Scale

Uses Tailwind's spacing scale consistently:

Token Value Usage
space-1 4px Inline padding, icon gaps
space-2 8px Tight element spacing
space-3 12px Button padding-x
space-4 16px Card internal gaps
space-6 24px Grid gutters, card padding
space-8 32px Section sub-headers
space-12 48px Major section gaps
space-16 64px Hero to content gap
space-20 80px Page section spacing
space-24 96px Major section spacing
space-32 128px Hero section height padding

Spacing Rules

  • Vertical rhythm: Use space-20 to space-24 between major page sections
  • Card padding: space-6 (24px) default, space-8 (32px) for featured cards
  • Text blocks: space-4 (16px) between paragraphs, space-8 (32px) before headings
  • Never use arbitrary spacing values — stick to the Tailwind scale

Layout Principles

  1. One dominant element: Every section has a clear visual hierarchy
  2. Asymmetric balance: Content and whitespace are intentionally uneven
  3. Breathing room: When in doubt, add more space
  4. Content-first: Layout serves readability, not decoration

Buttons

Primary

Padding: 16px 24px

Button Rules

  • Minimum touch target: 44x44px (accessibility)
  • Never more than 2 buttons side-by-side
  • Primary + Secondary pairing: Primary first (left/top)
  • Icon buttons: 40x40px square, icon centered, same hover rules

11. Voice & Copy

Writing Principles

1. Lead with the Result

The value proposition comes first. Setup comes second.

YES: "Deploy 3 AI agents in 30 minutes with ACOS."
NO:  "ACOS is a comprehensive operating system that enables..."

YES: "This workflow publishes 3 articles per week."
NO:  "In today's fast-paced content landscape..."

2. Be Specific, Not Impressive

Numbers over adjectives. Evidence over claims.

YES: "500+ AI songs generated. 70+ blog posts published. 40+ agents orchestrated."
NO:  "Revolutionary AI-powered creative platform."

YES: "Reduces agent setup from 4 hours to 30 minutes."
NO:  "Dramatically accelerates your workflow."

3. Studio Energy

Write like you're explaining to a friend at 2am. Direct, engaged, no performance.

YES: "Here's the exact pattern I use for multi-agent coordination."
NO:  "In this comprehensive guide, we'll explore the paradigms of..."

YES: "This breaks in production. Here's the fix."
NO:  "Users may experience challenges when deploying to production environments."

4. Technical Precision

Use correct terminology. Don't simplify past accuracy.

YES: "Server Components render on the server and send HTML to the client."
NO:  "Server Components make your app faster."

YES: "MCP servers expose tools via JSON-RPC over stdio."
NO:  "MCP connects AI to external tools."

5. Action Over Philosophy

Every piece ends with "what to do next."

YES: "Get the template → npm create frankx-app"
NO:  "Join us on this journey of discovery."

Headline Formulas

For Blog Posts

[Action] + [Specific Thing] + [With What]
"Build Multi-Agent Systems with Claude Code and MCP"
"Deploy AI Pipelines on Oracle Cloud in 30 Minutes"

[Number] + [Things] + [Outcome]
"7 Patterns for Production-Grade AI Agents"
"5 TypeScript Tricks That Eliminated 90% of Runtime Errors"

For Product Pages

[Outcome] + [How]
"Ship AI Products Faster with ACOS"
"Master Your Creative State with Vibe OS"

[Imperative] + [What]
"Build what matters."
"Orchestrate intelligence."

For Social Media

[Surprising Fact/Number] + [Context]
"I published 70+ blog posts in 30 days. Here's the system."
"500 AI songs. Zero music theory. Full creative control."

13. Motion & Animation

Principles

  1. Purposeful: Every animation communicates something (entry, feedback, transition)
  2. Fast: 150-300ms for interactions. Never exceed 500ms.
  3. Subtle: If you notice the animation, it's too much
  4. Respectful: Always honor prefers-reduced-motion

Timing

Duration Usage
100ms Color changes, opacity toggles
150ms Hover states, button feedback
200ms Card lifts, menu opens
300ms Page transitions, modal entry
500ms Complex orchestrated sequences (max)

Easing

/* Default (most interactions) */
transition-timing-function: cubic-bezier(0.4, 0, 0.2, 1);

/* Enter (elements appearing) */
transition-timing-function: cubic-bezier(0, 0, 0.2, 1);

/* Exit (elements leaving) */
transition-timing-function: cubic-bezier(0.4, 0, 1, 1);

/* Spring (playful interactions) */
transition-timing-function: cubic-bezier(0.34, 1.56, 0.64, 1);

Standard Animations

Hover Lift

transform: translateY(-2px);
box-shadow: 0 8px 24px rgba(0, 0, 0, 0.2);
transition: all 200ms cubic-bezier(0.4, 0, 0.2, 1);

Fade In Up (Page entry)

@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(16px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
animation: fadeInUp 300ms cubic-bezier(0, 0, 0.2, 1);

Staggered Entry

Delay each child by 50ms for list/grid items. Maximum stagger: 5 items (250ms total). Beyond that, fade the rest in together.

Reduced Motion

@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}

All Framer Motion animations must include layout prop considerations and prefers-reduced-motion checks.

14. Accessibility

Standards

FrankX targets WCAG 2.2 Level AA compliance, with AAA where practical.

Requirements

Color

  • Minimum contrast: 4.5:1 for normal text, 3:1 for large text
  • Color is never the only indicator (always pair with icon/text)
  • Dark mode is primary, light mode must also meet standards

Keyboard

  • All interactive elements reachable via Tab
  • Visible focus indicators (Aurora Blue ring, 3px)
  • Escape closes modals/menus
  • Arrow keys navigate within components

Screen Readers

  • All images have descriptive alt text
  • Form inputs have associated <label> elements
  • ARIA landmarks on all pages (nav, main, footer)
  • Dynamic content updates use aria-live regions

Motion

  • prefers-reduced-motion respected globally
  • No auto-playing video/audio
  • No flashing content (>3 flashes/second)

Touch

  • Minimum touch target: 44x44px
  • Adequate spacing between touch targets (8px minimum)
  • No hover-only interactions on mobile

Testing Checklist

  • Run axe-core in browser DevTools
  • Tab through entire page
  • Test with screen reader (NVDA or VoiceOver)
  • Verify at 200% zoom
  • Check prefers-reduced-motion behavior
  • Validate contrast ratios for all text
