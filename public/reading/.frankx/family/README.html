Family Relationship Graph

Schema for the FrankX family knowledge base. Each .md file represents one person with YAML frontmatter defining their node in the relationship graph.

Schema

---
id: unique-kebab-case-id # e.g. "frank-riemer"
name: Full Name
born_name: Name at birth # optional, for married name changes
born: YYYY or YYYY-MM-DD # birth year/date (if known)
location: Current city # optional
living: true/false
language: de | en | nl # primary language
relations:
  - type: spouse | parent | child | sibling | partner | grandparent | grandchild
    person: other-person-id
    note: optional context # e.g. "maternal grandfather"
tags: [grandparent, gorte-side] # freeform tags for filtering
---

Relation Types

Type Meaning Inverse
spouse Married to spouse
partner Life partner partner
parent Is parent of child
child Is child of parent
sibling Brother/sister sibling
grandparent Is grandparent of grandchild
grandchild Is grandchild of grandparent

Querying

Any AI agent can:

  1. Read all .md files in this directory
  2. Parse YAML frontmatter to build an in-memory graph
  3. Traverse relations to find connections
  4. Use tags for filtering (e.g. all gorte-side members)
  5. Reference in blog posts, pages, or content generation
