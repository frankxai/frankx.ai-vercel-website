Performance Guardian Analysis | Core Web Vitals & Speed Optimization

Date: January 14, 2026 Site: https://frankx.ai Framework: Next.js 16.1.1

Executive Summary

Current Performance Score: 6.5/10

The FrankX.AI website demonstrates moderate performance with several critical optimization opportunities.

Critical Issues Identified

  • 583MB public directory with unoptimized images
  • 595MB node_modules with heavy dependencies
  • 67 client components causing excessive hydration overhead
  • 3 PNG images > 1.3MB each blocking LCP
  • Minimal Next.js Image component usage (only 5 instances)
  • No bundle analyzer configured

1. Core Web Vitals Analysis

1.1 LCP (Largest Contentful Paint)

Target: < 2.5s | Current Estimate: 4.0-5.5s ⚠️

Critical Issues:

  1. Unoptimized Hero Images

    • /public/images/blog/agentic-roadmap-2025.png - 1.3MB
    • /public/images/blog/golden-age-intelligence.png - 1.4MB
    • /public/images/blog/music-consciousness.png - 1.5MB

  2. Font Loading Strategy

    • Loading 5 font weights (300, 400, 500, 600, 700)
    • Should reduce to 3 weights (400, 600, 700)

2. Specific File-Level Recommendations

HIGH PRIORITY

/app/layout.tsx (Lines 14-34)

// CURRENT - Loading 5 weights
const inter = Inter({
  weight: ['300', '400', '500', '600', '700'],
})

// RECOMMENDED - Reduce to 3 weights
const inter = Inter({
  weight: ['400', '600', '700'],
  preload: true,
})

Impact: -120KB font payload, -0.3s LCP

/components/home/OptimizedHomePage.tsx (Lines 1-6)

// ISSUE 1: Unnecessary 'use client' at top level
'use client' // ❌ Remove this

// ISSUE 2: Synchronous framer-motion import
import { motion } from 'framer-motion' // ❌ 50KB loaded immediately

// FIX: Lazy load animation library
const motion = dynamic(() => import('framer-motion').then(m => ({ default: m.motion })))

Impact: -50KB initial bundle, -200ms FID

/public/images/blog/*.png (3 files)

# Convert to WebP
npx @squoosh/cli --webp '{"quality":85}' public/images/blog/*.png

# Or use sharp for batch processing
npm install sharp
node scripts/optimize-images.js

Impact: -3.2MB payload, -2.5s LCP
